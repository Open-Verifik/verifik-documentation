# インド — EPIC（Voter ID）投票・投票所情報

このエンドポイントは、有効な **10 文字 EPIC**（*Elector's Photo Identity Card*）に基づき、有権者が **どこで投票するよう割り当てられているか** — 議会区、地区、部（part）、建物名および関連する場所フィールド — を返します。物流・サポート・コンプライアンスで **投票所コンテキスト** が必要な場合、[氏名照会 API](/verifik-ja/legal/epic-eci) を補完します。

## この API は何を検証しますか？

API は指定 EPIC の選挙 **場所・割当** データを返します。例:

- **選挙区と部**: 議会区（AC）識別子、部番号・名称、該当する場合は通し番号。
- **地理**: ソースが公開する地区、州、議会関連情報。
- **投票所**: EPIC に対して返される建物名、住所、部屋・ブース詳細。
- **EPIC 一致**: `documentNumber` に完全で有効な **10 文字** EPIC を使用する必要があります。

## API リファレンス

### Endpoint

```
https://api.verifik.co/v2/in/epic/votacion
```

`documentNumber` に EPIC を指定して **GET** を送信します。レスポンスは **投票場所** データ向けです（`GET /v2/in/epic` の氏名のみプロファイルではありません）。

### Headers

| 名前          | 値                 |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### Parameters

| 名前             | 型     | 必須     | 説明                                                                            | 例          |
| ---------------- | ------ | -------- | ------------------------------------------------------------------------------- | ----------- |
| `documentNumber` | string | **はい** | EPIC / Voter ID。**ちょうど 10 文字**（発行どおりの英数字）である必要があります。 | `IHM2796746` |

### Notes

- `documentNumber` は **ちょうど 10 文字** である必要があります。無効な長さや形式では `409` が返る場合があります。
- フィールドの有無は州と ECI データの可用性により異なり、任意の文字列に `null` が現れることがあります。
- **有権者名と親族名のみ** が必要な場合は [**GET /v2/in/epic**](/verifik-ja/legal/epic-eci) を使用してください。

## 一般的なユースケース

- **有権者ヘルプデスク**: EPIC に対応する正しい投票所・AC・建物へ案内。
- **物流・現地オペレーション**: 派遣や対面サポート前に割当を事前検証。
- 照会時点で EPIC がどの選挙地理に対応するかを示す必要がある **規制・監査ワークフロー**。

## 公式ソースと信頼性

Verifik は ECI 指向の選挙データを正規化し、単一の API 契約を提供します。結果は照会時点の公式レジストリに依存します。無効、または現行ソースから返されない EPIC については常に `404` を処理してください。

## 主なメリット

- **エンドツーエンド TLS** と、成功時の Verifik `signature` 付きの一貫した JSON 形。
- **氏名照会の補完**: 氏名と場所の両方が必要な KYC スタイルの EPIC 確認では `/v2/in/epic` と組み合わせます。

## コンプライアンスとセキュリティ

`Authorization` トークンを保護し、公開アナリティクスに生の EPIC 値を **絶対に記録しないでください**。選挙データの保持はポリシーと現地法の範囲内に保ってください。Verifik は転送セキュリティとプラットフォームレベルの監視を適用します。
