# アルゼンチンの本人確認（拡張）

Verifik の本人確認 API は公的な政府データでアルゼンチン市民を認証します。本エンドポイントは標準 cédula 応答を超える拡張フィールド（**生年月日**、**性別**、構造化された姓名、**生存状態** `isAlive`）を返します。

## この API は何を検証しますか？

本 API は公式記録と直接接続し、以下を検証します。

-   **氏名と DNI 番号**: *Documento Nacional de Identidad*（DNI）、文書タイプ `DNIAR` に対応。
-   **本人一致**: 文書番号が登録者に対応するかを確認。
-   **追加データ**: **dateOfBirth**、**gender**、**firstName**、**lastName**、**fullName**、**arrayName**（姓名パーツ）、**isAlive** を返します。

これらを検証することで、相手が実在し有効な文書を保有していることを確認でき、なりすましや不正のリスクを大幅に低減できます。

## API リファレンス

### エンドポイント

```
https://api.verifik.co/v2/ar/cedula/extra
```

### ヘッダー

| 名前          | 値                 |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### パラメーター

| 名前             | 型     | 必須 | 説明                                                                 |
| ---------------- | ------ | -------- | --------------------------------------------------------------------------- |
| `documentType`   | string | はい | 文書タイプ。許可値: `DNIAR`。                                      |
| `documentNumber` | string | はい | 照会する DNI 番号（スペースなし）。                                        |

### 注意

- `documentType` は `DNIAR` である必要があります。
- 利用可能な場合、`gender` は 1 文字コード（例: `F` または `M`）で返されます。
- `isAlive` は照会時点で登録上その人物が生存しているかを示します。

## 一般的なユースケース

-   **フィンテック・銀行**: 生年月日と生存状態チェックでオンボーディングを強化。
-   **EC・配送**: 配送員や高額アカウント保有者をより強い本人信号で検証。
-   **人事・採用**: 公式記録と照合して候補者の本人詳細を確認。
-   **保険・ヘルスケア**: 検証済み生年月日で資格・年齢関連ルールを支援。

## 公式ソースと信頼性

アルゼンチンの公的政府ソースと直接接続し、検証済みかつ最新の情報を提供します。
すべての照会は厳格なセキュリティおよび規制基準に基づいて処理されます。

## 主な利点

-   **より豊富な本人信号**: 氏名・文書番号に加え生年月日、性別、生存状態。
-   **自動化されたコンプライアンス**: ユーザー体験を損なわず KYC を効率化。
-   **即時の結果**: 数秒で処理し、リアルタイム onboarding に最適。
-   **信頼できるデータ**: 公式記録から直接取得。
-   **容易な統合**: REST API または互換 SDK で接続。

## コンプライアンスとセキュリティ

データの安全性を最優先。Verifik は HTTPS/TLS 1.3 と厳格なプライバシー基準を採用。
可用性を 24/7 監視し、ロールベースのアクセス制御を提供。

## Verifik について

Verifik はラテンアメリカ全域の本人確認・コンプライアンス・不正防止のリーディングプラットフォームです。
当社 API は KYC、KYB、AML、生体検証を自動化し、各国の公式データソースとビジネスを接続します。
