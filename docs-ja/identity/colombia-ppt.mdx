### エンドポイント

```
GET https://api.verifik.co/v2/co/foreigner-id/ppt
```

**`documentNumber`** と発行日 **`expeditionDate`**（`DD/MM/YYYY`）が必要です。**`POST`** でも同じフィールドを JSON で送れます。

### ヘッダー(Headers)

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### 書類要件

**対象:** **Migración Colombia** から **入国ステータス**（VIGENTE、有効期限など）が必要な *Permiso de Protección Temporal*（**PPT**）保有の **ベネズエラ国民**。

| フィールド | ガイド |
| --- | --- |
| **`documentNumber`** | PPT 番号、**数字のみ**。コロンビアでは通常 **最大 7 桁**。一部システム（SENA、給与）は **15 桁**パディング — 記録の数値を送信。Migración 例：`2081381`。 |
| **`expeditionDate`** | 発行日 **`DD/MM/YYYY`**（例：`07/03/2022`）。必須。 |

**別途:** `documentType=PPT` で [`/v2/co/cedula`](/verifik-ja/identity-validation/colombia/colombia-identity-documents-guide) から氏名型照会（5–10 桁検証、発行日不要）。

一覧比較：[身分証明書ガイド](/verifik-ja/identity-validation/colombia/colombia-identity-documents-guide)。

### パラメータ(Parameters)

| name | type | required | description |
| --- | --- | --- | --- |
| `documentNumber` | string | yes | PPT 番号、**数字のみ**。通常 **最大 7 桁**（15 桁パディングのシステムあり）。例：`2081381`。 |
| `expeditionDate` | string | yes | 発行日 **`DD/MM/YYYY`**（例：`07/03/2022`）。書類と一致させる。 |

### リクエスト(Request)

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/co/foreigner-id/ppt", {
	params: { documentNumber: "123456789", expeditionDate: "07/03/2022" },
	headers: {
		Accept: "application/json",
		Authorization: `Bearer ${process.env.VERIFIK_TOKEN}`,
	},
});
console.log(data);
```

### 備考(Notes)

- 日付は **`DD/MM/YYYY`** 形式です。
- **`GET`** / **`POST`** は同じ処理です。
- 上流の不調で **404** / **500** になることがあります。
