# 🇧🇷 巴西 — 车辆信息

## 车辆信息查询

### Endpoint

```
GET https://api.verifik.co/v2/br/vehicle
```

给定巴西**车牌**，**Verifik** 在一次请求中返回结构化车辆数据，便于将实物资产与已验证、已规范化的属性关联。适用于**车队与出行入驻**、**保险报价与理赔**、**担保融资与抵押审核**，以及以车牌为主要标识的**市场或经销商核验**。

Verifik API 返回该车牌对应的**规范化车辆记录**，包括**车牌**、**品牌**、**型号**、**车型年款**与**制造年份**、**底盘（VIN）**、**发动机**、**颜色**、**车身类型**、**燃料**、**变速箱**、**车门数**、**制造商或工厂**、适用于该车辆的 **FIPE 代码**，以及该登记存在时的**违规或限制指标**。用于在继续流程前确认车辆与官方信息一致。

### 请求头（Headers）

| 名称           | 取值                |
| -------------- | ------------------- |
| Accept         | `application/json`  |
| Authorization  | `Bearer <token>`    |

### 参数（Parameters）

| 名称 | 类型 | 必填 | 描述 |
| ---- | ---- | ---- | ---- |
| `plate` | string | **是** | 待查询车牌（字母与数字；大小写会规范化）。示例：`PZY7H82`。 |

### 响应字段

| 字段 | 类型 | 描述 |
| ---- | ---- | ---- |
| `bodyType` | string | 车辆登记来源返回的车辆类型标签。 |
| `brand` | string | 车辆品牌。 |
| `chassis` | string | 登记来源返回的底盘号或 VIN。 |
| `color` | string | 登记的车辆颜色。 |
| `country` | string | 车辆记录所属国家。 |
| `denatranWarning` | string | Denatran 或登记状态提示，例如 `Sem Restrição`。 |
| `doors` | string | 登记来源返回的乘客容量。 |
| `engine` | string | 登记中的排量或发动机参考值。 |
| `factory` | string | 来源或制造地区。 |
| `fipeCodes` | string[] | 可用时返回与车辆相关的 FIPE 参考代码。 |
| `fuelType` | string | 登记的燃料类型。 |
| `irregularitiesCount` | number | 找到的有效限制或异常数量。 |
| `irregularityCode` | string | 有效限制标签合并后的字符串；没有限制时为空。 |
| `manufacturer` | string | 登记来源返回的制造商名称。 |
| `model` | string | 登记的车型描述。 |
| `modelYear` | string | 车型年份。 |
| `plate` | string | 查询返回的规范化车牌。 |
| `transmission` | string | 可用时返回变速箱信息；登记来源记录可能返回空字符串。 |
| `vehicle` | string | 登记中的车辆类别。 |
| `version` | string | 可用时返回最佳 FIPE 车型/版本匹配。 |
| `yearOfManufacture` | string | 制造年份。 |

### 备注

- 查询字符串中**必填**参数仅为 **`plate`**。
- 登记来源记录可能将 **`transmission`** 返回为空字符串。
- **`doors`** 表示登记来源中的乘客容量。
- 没有 FIPE 匹配时，**`fipeCodes`** 可能为空数组。
- 当主来源没有匹配并使用备用提供方时，返回字段覆盖范围可能略有不同。
- 部分账户还会启用**按车牌扩展技术表**产品；启用时请使用 AppFeature 目录中给出的 URL。
