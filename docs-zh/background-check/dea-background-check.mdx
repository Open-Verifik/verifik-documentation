# DEA 背景核查

## API 参考

### Endpoint

```
https://api.verifik.co/v2/dea
```

使用 `documentType` 与 `documentNumber`，或直接使用 `fullName`，对个人或机构在美国缉毒局（DEA）观察名单上进行筛查。该 API 会返回是否命中匹配，并在适用时返回指向 DEA 官方记录的 URL。

**注意：** 日期字段（出生日期或签发日期）必须采用 `dd/mm/yyyy` 格式。

### 请求头（Headers）

| 名称           | 取值                |
| -------------- | ------------------- |
| Content-Type   | `application/json`  |
| Authorization  | `Bearer <token>`    |

### 参数（Parameters）

:::info
有关支持的**证件类型**及其专属必填参数的完整清单，请参阅 [证件类型](/background-check/document-types) 指南。
:::

| 名称              | 类型    | 必填 | 描述                                                                                              |
| ----------------- | ------- | ---- | ------------------------------------------------------------------------------------------------- |
| `documentType`    | string  | 否   | 待请求的证件类型。                                                                                 |
| `documentNumber`  | string  | 否   | 待查询的证件号码，不含空格或圆点。                                                                 |
| `fullName`        | string  | 否   | 可不传 documentType 与 documentNumber，而直接传入个人或企业的姓名/名称。                            |

### 使用场景

- **监管合规：** 依据美国联邦法规筛查与 DEA 相关的风险标记。
- **欺诈预防：** 识别与管制物质执法记录相关的风险敞口。
- **KYC 与 AML：** 服务于医疗、制药、物流与金融行业的合规项目。
