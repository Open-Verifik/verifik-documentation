# 哥伦比亚 — RUES 企业查询 (v3)

Verifik 通过 **`NIT`**、**登记类别（category）** 查询哥伦比亚 **RUES**（*Registro Único Empresarial y Social*），返回该类别下的 **标准化基础记录**（商业登记、RUP 投标人、旅游、非营利等）。本接口为较轻量的 **`/v3/co/rues`**；如需 **完整** 企业档案（法定代表人、经济活动、分支机构等），请使用 **`GET /v3/co/rues-complete`**。

## 应选用哪个接口？

| | **`GET /v3/co/rues`**（本接口） | **`GET /v3/co/rues-complete`** |
| --- | --- | --- |
| **用途** | 按登记类别快速 **摘要** | **完整** 企业档案 |
| **典型字段** | 类别核心字段（名称、NIT、状态、商会等） | 含法定代表人与经济活动的完整登记块 |
| **参数** | `documentType`（**`NIT`**）、`documentNumber`、可选 `category` | **相同** |
| **目录定价** | 较低（如 **0.2** 积分 / **0.3** SmartCheck） | 较高（如 **0.3** 积分 / **0.4** SmartCheck） |

完整档案说明见 [RUES 企业完整核验 (v3)](/business-validation/rues-complete-v3)。

## 本接口返回内容

- 按 **类别** 的摘要字段（如 **RM**：商号、含校验位的 NIT、商会、登记状态、组织类型等，以实际响应为准）
- 经集成所用 RUES 响应 schema 映射的数据
- 可能启用 **缓存**
- 经 Verifik **签名** 的响应

## API 参考

### 端点

```
GET https://api.verifik.co/v3/co/rues
```

本路由仅支持 **GET** 与 **查询参数**。完整档案请用 **`GET /v3/co/rues-complete`**（参数相同）。

查询参数：**`documentType`**（仅 **`NIT`**）、**`documentNumber`**、可选 **`category`**。

应用域名：`GET https://verifik.app/v3/co/rues`

### 请求头

| 名称 | 值 |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### 参数

| 名称 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `documentType` | string | 是 | 必须为 **`NIT`**。 |
| `documentNumber` | string | 是 | 税号数字；服务端会去除标点。 |
| `category` | string | 否 | RUES 登记线：**`RM`**, **`PROP`**, **`RUNEOL`**, **`RNT`**, **`ESAL`**, **`ESOL`**, **`JUEGOS`**, **`EXTRANJERAS`**。省略时默认为 **`RM`**。 |

## 特性

- API 层 **仅支持 NIT** 校验
- 可选 **类别**：RM、PROP、旅游 (RNT)、ESAL 等
- **v3** RUES elastic API 集成（各类别行为见模块说明）

## 使用场景

- **KYB** 与 **供应商** 筛查
- 拉取完整 RUES Complete 前的 **登记确认**

### 说明

- **`category`** 在处理器中为可选（**默认 `RM`**）；非 **Registro Mercantil** 查询时请显式传入。
- 本接口返回 **摘要**，非完整 RUES 档案。法定代表人、经济活动、分支机构等请用 **`GET /v3/co/rues-complete`**（[RUES 完整核验 (v3)](/business-validation/rues-complete-v3)）。
- 校验接受的某些类别代码在模块内可能走 **旧版** 路径；若结果异常，请与支持确认类别或尝试 **`RM`** / **`rues-complete`**。
- 旧集成可使用 **`/v2/co/rues`**；新项目请优先 **v3**。
- RUES 可用性及 captcha/限流可能导致 **404** / **409** / **5xx**，取决于上游状态。
