# 厄瓜多尔企业核验

Verifik 的企业核验 API 可基于政府官方数据对厄瓜多尔企业进行身份核实。该接口旨在简化您的 KYB（了解您的企业）流程、防范欺诈，并满足合规要求。

我们专为需要快速、安全且自动化确认企业、合作伙伴或供应商合法性的组织打造此集成。

## 此 API 验证哪些内容？

我们的 API 直接对接官方记录，验证以下内容：

-   **商号与 RUCEC**：支持 *Registro Único de Contribuyentes del Ecuador*（RUCEC）。
-   **企业信息**：返回商号、状态、纳税人类型、税制、主要经济活动、会计与预扣标志、纳税人日期及法定代表人。
-   **企业匹配**：确认企业信息与 RUCEC 号码相符。

通过验证这些信息，您可以降低欺诈风险并支持合规工作流。

## API 参考

### Endpoint

```
GET https://api.verifik.co/v3/ec/company
```

使用 **`documentType`** `RUCEC` 与 **`documentNumber`**（13 位数字，无分隔符）查询厄瓜多尔企业。返回面向登记的字段——包括经济活动、税制与法定代表人——以支持 **KYB**、**供应商筛查** 与 **合规**。

### 请求头（Headers）

| 名称           | 取值                |
| -------------- | ------------------- |
| Content-Type   | `application/json`  |
| Authorization  | `Bearer <token>`    |

### 参数（Parameters）

| 名称              | 类型   | 必填 | 描述                                         | 示例            |
| ----------------- | ------ | ---- | -------------------------------------------- | --------------- |
| `documentType`    | string | 是   | 证件类型。允许取值：`RUCEC`。                | `RUCEC`         |
| `documentNumber`  | string | 是   | 13 位 RUCEC，不含空格或句点。                | `1793197955001` |

### Notes

- `documentNumber` 必须恰好为 13 位数字，且不含分隔符。
- 沙箱演示 RUC：`1790008959001`（ACERO COMERCIAL ECUATORIANO S.A.）。
- 线上示例：`1793197955001`、`1191795453001`。
- 旧版 `GET /v2/ec/company` 仍可用，但响应结构更精简。

## 常见使用场景

-   **B2B 入驻**：在合作伙伴或供应商注册时即时核验企业。
-   **金融服务**：在开户或处理交易前对企业进行身份核实。
-   **合规与尽职调查**：在 KYC/KYB 工作流中验证企业信息。
-   **电商与市场平台**：在允许企业上架销售前确认其合法性。
