# 玻利维亚 - 投票地点

Verifik 使用 **CI** 和**出生日期**查询 **Yo Participo (OEP)**。在允许使用 OEP 数据的场景下，可用于**选举核验**、**选民 onboarding** 和**合规**流程。

在数据源提供时，返回选举地点详情：

- **`documentType`** — 允许值：**CI**
- **`documentNumber`** — 玻利维亚身份证号（**5–13** 位）
- **`dateOfBirth`** — **DD/MM/YYYY** 格式的出生日期（须与 OEP 记录一致）
- **`fullName`** — 选举登记中的公民全名
- **`votingStatus`** — 公民是否具备投票资格
- **`department`**、**`locality`**、**`pollingPlace`**、**`pollingTable`** — 选举地点字段
- **`election`** — 提供时的选举标签或登记更新说明
- **可选字段** — 数据源包含时：`isJury`、`jurySortDate`、`politicalMembership`、`latitude`、`longitude`
- **`force`** — 可选；`true` 绕过缓存
- Verifik **签名** 响应

### Endpoint

```
GET v2/bo/votacion
```

### Headers

| Header | Value | Description |
| --- | --- | --- |
| Accept | application/json | 响应格式 |
| Authorization | Bearer \<token\> | 您的 Verifik JWT |

### Parameters

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| documentType | String | Yes | 证件类型。允许值：`CI`。 |
| documentNumber | String | Yes | 无签发代码的玻利维亚 CI 号（**5–13** 位）。 |
| dateOfBirth | String | Yes | `DD/MM/YYYY` 格式的出生日期。 |
| force | String | No | 设为 `true` 可绕过缓存并获取最新数据。 |

### Notes

- **`documentType`** 必须为 **`CI`**；**`dateOfBirth`** 为必填且须与 OEP 记录一致。
- 可选的陪审员、党籍和坐标字段仅在数据源 HTML/JSON 包含时出现。
- 需要绕过 MongoDB 缓存记录时使用 **`force=true`**。
- 响应包含**敏感个人数据**；请遵守适用的隐私法规。
