# 印度 — EPIC（选民证）姓名查询

Verifik 提供单一 REST 端点，用于查询与印度 **EPIC**（*Elector's Photo Identity Card*）关联的 **选民姓名** 信息，数据来源与 **印度选举委员会（ECI）** 对齐。适用于需要确认选民证号码返回一致登记姓名的身份核验、入驻与合规流程。

## 此 API 验证什么？

API 连接官方选举数据并返回：

- **选民姓名结构**：名与姓（在可用时包含本地文字变体）。
- **全名与亲属姓名**：适合以拉丁字母与本地文字显示与匹配的字段。
- **EPIC 匹配**：确认所提供的 **10 位** EPIC（字母与数字）被识别并返回相关记录。

这可减少人工核查，并使选举相关验证与底层登记册保持一致。

## API 参考

### Endpoint

```
https://api.verifik.co/v2/in/epic
```

在查询参数 `documentNumber` 中发送 EPIC，发起 **GET** 请求。成功时服务返回结构化姓名字段、可选本地文字值，以及 Verifik **signature** 块。

### Headers

| 名称          | 值                 |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### Parameters

| 名称             | 类型   | 必填     | 说明                                                                  | 示例        |
| ---------------- | ------ | -------- | --------------------------------------------------------------------- | ----------- |
| `documentNumber` | string | **是**   | EPIC / 选民证。必须为 **恰好 10 个字符**（按签发的字母数字）。     | `IHM2796746` |

### Notes

- `documentNumber` 必须为 **恰好 10 个字符**；长度或格式无效可能返回 `409`。
- 数据 **通过 Verifik** 获取；覆盖范围遵循该 EPIC 的 ECI 可用性。
- 如需 **投票站与选区位置** 详情，请使用配套端点 [**印度 — EPIC 投票/投票站信息**](/verifik-zh/legal/voting-eci-epic)（`GET /v2/in/epic/votacion`）。

## 常见用例

- 需要用返回的姓名与亲属姓名核验选民证的 **KYC 与身份项目**。
- 在发放凭证或访问前必须验证 EPIC 的 **政府与市民服务**。
- 选举身份检查构成更广泛风险与欺诈策略一部分的 **金融科技与合规**。

## 官方来源与可靠性

Verifik 将请求路由至可信选举来源，使您获得与 ECI 公开查询行为一致的数据。所有调用在传输中使用 TLS；请在日志与存储策略中将 EPIC 视为 **敏感个人数据**。

## 主要优势

- **集成简单**：一次带 `documentNumber` 与标准 `Authorization` 头的 GET。
- **结构化响应**：在来源提供时包含姓名数组与本地文字字段。
- **Verifik 认证**：成功响应可包含用于审计追踪的 Verifik `signature` 块。

## 合规与安全

仅使用 HTTPS（上述 URL 受 TLS 保护）。安全存储令牌、轮换 API 密钥，并遵循您所在司法辖区对选举与身份数据的数据保护规则。Verifik 平台面向高可用与受监控访问而设计。
