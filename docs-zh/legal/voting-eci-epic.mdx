# 印度 — EPIC（选民证）投票与投票站信息

此端点返回选民 **被分配到何处投票**：议会选区、地区、分区（part）、建筑名称及相关位置字段——基于有效的 **10 位 EPIC**（*Elector's Photo Identity Card*）。当您需要 **投票站上下文** 用于物流、支持或合规时，它可补充 [姓名查询 API](/verifik-zh/legal/epic-eci)。

## 此 API 验证什么？

API 返回给定 EPIC 的选举 **位置与分配** 数据，例如：

- **选区与分区**：议会选区（AC）标识、分区编号与名称，以及适用时的序号。
- **地理**：来源暴露时的地区、邦与议会关联信息。
- **投票站**：为该 EPIC 返回的建筑名称、地址及房间/票亭详情。
- **EPIC 匹配**：查询必须在 `documentNumber` 中使用完整有效的 **10 位** EPIC。

## API 参考

### Endpoint

```
https://api.verifik.co/v2/in/epic/votacion
```

在 `documentNumber` 中发送 EPIC，发起 **GET** 请求。响应面向 **投票地点** 数据（而非 `GET /v2/in/epic` 返回的仅姓名资料）。

### Headers

| 名称          | 值                 |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### Parameters

| 名称             | 类型   | 必填     | 说明                                                                  | 示例        |
| ---------------- | ------ | -------- | --------------------------------------------------------------------- | ----------- |
| `documentNumber` | string | **是**   | EPIC / 选民证。必须为 **恰好 10 个字符**（按签发的字母数字）。     | `IHM2796746` |

### Notes

- `documentNumber` 必须为 **恰好 10 个字符**；长度或格式无效可能返回 `409`。
- 字段是否存在可能因邦与 ECI 数据可用性而异；可选字符串可能出现 `null`。
- 若仅需 **选民姓名与亲属姓名**，请使用 [**GET /v2/in/epic**](/verifik-zh/legal/epic-eci)。

## 常见用例

- **选民服务台**：将公民引导至对应 EPIC 的正确投票站、AC 与建筑。
- **物流与现场运营**：在派遣或当面支持前预验证分配。
- 需在查询时展示 EPIC 映射到何种选举地理的 **监管与审计流程**。

## 官方来源与可靠性

Verifik 规范化面向 ECI 的选举数据，提供单一 API 契约。结果取决于查询时的官方登记册；对无效或当前来源未返回的 EPIC，请始终处理 `404`。

## 主要优势

- **端到端 TLS**，以及成功时带 Verifik `signature` 的一致 JSON 形态。
- **补充姓名查询**：在需要姓名与地点时，与 `/v2/in/epic` 配对进行 KYC 风格的 EPIC 检查。

## 合规与安全

保护 `Authorization` 令牌，**切勿**在公开分析中记录原始 EPIC 值。将选举数据的保留控制在您的政策与当地法律范围内。Verifik 在 API 边缘强制传输安全与平台级监控。
