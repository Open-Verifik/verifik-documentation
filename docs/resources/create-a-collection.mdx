# Create a Collection

Create a new collection for storing faces generated using Verifik's Life Detection services. Collections are required for the majority of biometric services.

## Endpoint

```
POST https://api.verifik.co/v2/face-recognition/collections
```

## Headers

| Name          | Value              |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

## Parameters

| Name          | Type   | Required | Description                                    |
| ------------- | ------ | -------- | ---------------------------------------------- |
| `name`        | string | Yes      | Name of the collection.                       |
| `description` | string | Yes      | Brief description of what the collection will be used for. |

## Use Cases

- **Face Recognition Setup**: Create collections to store face data for biometric services
- **Multi-Project Management**: Organize face data by different projects or applications
- **Service Integration**: Prepare collections required for face comparison and search services
- **Data Organization**: Categorize face data for different use cases or user groups
- **API Development**: Set up collections for testing and development purposes

## Features

- ✅ **Unique Collection Code**: Each collection gets a unique code for API operations
- ✅ **Project Association**: Collections are automatically linked to your project
- ✅ **Client Management**: Collections are associated with your client account
- ✅ **Automatic Timestamps**: Creation and update timestamps are automatically managed
- ✅ **Soft Delete Support**: Collections can be marked as deleted without permanent removal
- ✅ **Multiple Programming Languages**: Support for JavaScript, Python, Swift, and PHP
- ✅ **Comprehensive Error Handling**: Detailed error responses for various scenarios
