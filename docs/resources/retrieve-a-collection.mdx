# Retrieve a Collection

Fetch a collection generated using its specific _id, not to be confused with the code parameter.

## Endpoint

```
GET https://api.verifik.co/v2/face-recognition/collections/{id}
```

## Headers

| Name          | Value              |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

## Parameters

| Name | Type   | Required | Description                                    |
| ---- | ------ | -------- | ---------------------------------------------- |
| `id` | string | Yes      | ID of the collection that you want to bring the information. |

## Use Cases

- **Collection Details**: Get detailed information about a specific collection
- **Service Integration**: Retrieve collection information for use with biometric services
- **Data Validation**: Verify collection exists and get its properties
- **API Development**: Test collection retrieval for development purposes
- **Monitoring**: Check specific collection status and metadata

## Features

- ✅ **Specific Collection**: Retrieve details for a single collection by ID
- ✅ **Complete Information**: Returns full collection details including code and timestamps
- ✅ **Project Association**: Shows which project the collection belongs to
- ✅ **Client Verification**: Ensures collection belongs to your client account
- ✅ **Multiple Programming Languages**: Support for JavaScript, Python, Swift, and PHP
- ✅ **Comprehensive Error Handling**: Detailed error responses for various scenarios
- ✅ **Authentication Required**: Secure access using Bearer token authentication
