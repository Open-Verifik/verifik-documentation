# Verifik public API — endpoint reference

This document explains, in plain language, **what each query** does to the API and **how to test it** with examples. You don't need to be a programmer: you can copy the examples into tools like Postman or ask someone technical to use them as is.

---

## Before you start

- **Base address (`UrlBase`)**: in Postman it is usually a variable; In the examples below we use `https://api.verifik.co` only as **example**. The real address is given by Verifik when you give you access.

- **Authentication**: many APIs ask for a **key** (token) in the header. If your collection uses it, you must replace `YOUR_KEY` with the key they give you.

- **Method**: indicates the type of operation. The usual thing is **GET** (only read data) or **POST** (send data in the body of the message).
