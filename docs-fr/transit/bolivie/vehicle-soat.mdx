# Bolivie - SOAT véhicule

Verifik interroge le registre officiel **SOAT** (*Seguro Obligatorio de Accidentes de Tránsito*) en Bolivie par **plaque d'immatriculation**. Utilisez ce point de terminaison pour la **conformité de flotte**, la **vérification d'assurance** et les **contrôles de transit**.

Retourne les détails de couverture SOAT lorsque la source les fournit :

- **`plate`** — plaque d'immatriculation (**5–8** caractères), normalisée en majuscules
- **`managementYear`** — année de gestion ou de couverture SOAT
- **`validFrom`** / **`validTo`** — dates de validité de la couverture (JJ/MM/AAAA)
- **`vehicleType`** — catégorie du véhicule (p. ex. AUTOMOVIL, VAGONETA, MOTOCICLETA)
- **`useType`** — classification d'usage (PARTICULAR, COMERCIAL)
- **`department`** — département associé à la police
- **`status`** — statut calculé : VIGENTE, VENCIDO ou PENDIENTE
- **`force`** — optionnel ; `true` ignore le cache et interroge la source
- Réponse **signée** par Verifik

### Endpoint

```
GET v2/bo/vehicle-soat
```

### Headers

| Header | Valeur | Description |
| --- | --- | --- |
| Accept | application/json | Format de réponse |
| Authorization | Bearer \<token\> | Votre JWT Verifik |

### Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| plate | String | Oui | Plaque d'immatriculation sans espaces (**5–8** caractères). |
| force | String | Non | Définir sur `true` pour ignorer le cache et interroger la source. |

### Notes

- **`plate`** est obligatoire ; le middleware la normalise en majuscules.
- **`status`** est calculé à partir de **`validFrom`** et **`validTo`** par rapport à la date actuelle.
- Utilisez **`force=true`** pour ignorer les enregistrements en cache MongoDB.
- Traitez les réponses comme des **données véhicule sensibles** ; respectez les lois applicables en matière de confidentialité.
