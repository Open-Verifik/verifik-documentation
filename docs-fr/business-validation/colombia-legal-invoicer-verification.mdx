# 🇨🇴 Colombie — Vérification facturier légal

L'API de vérification de facturier légal de Verifik vous aide à valider si une entreprise colombienne possède un facturier légal enregistré auprès de la DIAN (*Dirección de Impuestos y Aduanas Nacionales*). Elle est conçue pour fluidifier vos processus KYB (Know Your Business), prévenir la fraude et vous assurer de respecter sans effort toutes les exigences réglementaires.

Nous avons construit cette intégration pour les entreprises qui ont besoin d'un moyen rapide, sécurisé et automatisé de confirmer le statut d'enregistrement de facturier des entreprises, partenaires ou fournisseurs colombiens.

## Que valide cette API ?

Notre API se connecte directement aux registres officiels de la DIAN pour valider :

-   **NIT de l'entreprise et enregistrement de facturation électronique** : prend en charge le *Número de Identificación Tributaria* (NIT).
-   **Contact du facturier** : renvoie l'**e-mail** du facturier enregistré et les champs associés lorsque la DIAN les expose.
-   **Contrôle d'enregistrement** : confirme que l'entité apparaît dans le jeu de données **facturier légal** de la DIAN pour la facturation électronique (distinct du statut fiscal général — voir [Vérification DIAN](/verifik-fr/business-validation/colombia-dian-verification)).

En vérifiant ces éléments, vous pouvez avoir la certitude que l'entreprise dispose des capacités de facturation appropriées et est conforme aux exigences DIAN, ce qui réduit considérablement les risques de fraude et garantit la conformité.

## Référence API

### Point d'accès

```
GET https://api.verifik.co/v2/co/company/dian/invoicer
```

Même chemin sur l'application : `GET https://verifik.app/v2/co/company/dian/invoicer`.

### En-têtes

| Nom | Valeur |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Paramètres

| Nom | Type | Requis | Description |
| --- | --- | --- | --- |
| `documentType` | string | Oui | Doit être **`NIT`**. |
| `documentNumber` | string | Oui | NIF (NIT). Les espaces sont supprimés côté serveur ; vous pouvez inclure ou omettre le chiffre de vérification. Longueur minimale **5** caractères (validation API). |

### Notes

- **`GET` en query uniquement** — pas de corps. Utilisez **`Accept`** (pas `Content-Type`) pour les réponses JSON.
- Cette route renvoie les données d'**enregistrement de facturier légal / facturation électronique** (p. ex. **e-mail** lorsqu'il est disponible). Pour le **statut fiscal général** et les obligations, utilisez **[Vérification DIAN](/verifik-fr/business-validation/colombia-dian-verification)** (`GET /v2/co/company/dian`).
- **404** peut survenir lorsque la DIAN ne renvoie pas l'**e-mail** du facturier pour ce NIT (voir la logique d'intégration).
