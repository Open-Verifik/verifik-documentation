# 🇨🇴 Colombie — Vérification DIAN

L'API de vérification DIAN de Verifik vous aide à authentifier le statut fiscal des entreprises colombiennes à partir de données gouvernementales officielles. Elle est conçue pour fluidifier vos processus KYB (Know Your Business), prévenir la fraude et vous assurer de respecter sans effort toutes les exigences réglementaires.

Nous avons construit cette intégration pour les entreprises qui ont besoin d'un moyen rapide, sécurisé et automatisé de confirmer le statut fiscal et la légitimité d'entreprises, partenaires ou fournisseurs colombiens.

## Que valide cette API ?

Notre API se connecte directement aux registres officiels de la DIAN (*Dirección de Impuestos y Aduanas Nacionales*) pour valider :

-   **Raison sociale et NIT** : prend en charge le *Número de Identificación Tributaria* (NIT).
-   **Statut fiscal** : renvoie le statut d'enregistrement de l'entreprise auprès de la DIAN.
-   **Correspondance d'entreprise** : confirme que les informations correspondent au NIT et valide les responsabilités fiscales actives.

En vérifiant ces éléments, vous pouvez avoir la certitude que l'entreprise avec laquelle vous traitez est légitime, correctement enregistrée auprès de l'autorité fiscale et possède des obligations fiscales actives, ce qui réduit considérablement les risques de fraude et garantit la conformité.

## Référence API

### Point d'accès

```
GET https://api.verifik.co/v2/co/company/dian
```

Même chemin sur l'application : `GET https://verifik.app/v2/co/company/dian`.

| Nom | Valeur |
| --- | --- |

| Authorization | `Bearer <token>` |

### Paramètres

| Nom | Type | Requis | Description |
| --- | --- | --- | --- |
| `documentType` | string | Oui | Doit être **`NIT`**. |
| `documentNumber` | string | Oui | NIF (NIT). Les espaces sont supprimés côté serveur ; vous pouvez inclure ou omettre le chiffre de vérification. Longueur minimale **5** caractères (validation API). |

### Notes

- **`GET` en query uniquement** — pas de corps. Utilisez **`Accept`** (pas `Content-Type`) pour les réponses JSON.
- Utilisez une chaîne NIT valide ; l'intégration gère le formatage et le chiffre de vérification tel que renvoyé par la DIAN.
- **Découpage du nom pour les personnes naturelles** : lorsque la réponse concerne une « persona natural », Verifik découpe automatiquement le nom du champ `nombreRazon` en `firstName`, `lastName`, `fullName` et `arrayName`. Comme la source renvoie le nom dans un seul champ (`nombreRazon`), le découpage peut ne pas être exact à 100 %. Le champ `nombreRazon` contient toujours le nom original tel que renvoyé par la DIAN.
