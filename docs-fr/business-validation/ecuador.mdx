# 🇪🇨 Équateur — Vérification d'entreprise

L'API de vérification d'entreprise de Verifik vous permet d'authentifier les sociétés équatoriennes à partir des données officielles du gouvernement. Elle est conçue pour fluidifier vos processus KYB (Know Your Business), prévenir la fraude et vous assurer de respecter les exigences réglementaires.

Nous avons construit cette intégration pour les entreprises qui ont besoin d'un moyen rapide, sécurisé et automatisé de confirmer la légitimité de sociétés, partenaires ou fournisseurs.

## Que valide cette API ?

Notre API se connecte directement aux registres officiels pour valider :

-   **Raison sociale et RUCEC** : prend en charge le *Registro Único de Contribuyentes del Ecuador* (RUCEC).
-   **Informations sur l'entreprise** : retourne la raison sociale, le statut, le type de contribuable, le régime, l'activité économique principale, les indicateurs de comptabilité et de retenue, les dates du contribuable et les représentants légaux.
-   **Correspondance d'entreprise** : confirme que les informations correspondent au numéro RUCEC.

En vérifiant ces éléments, vous pouvez réduire les risques de fraude et soutenir vos flux de conformité.

## Référence de l'API

### Endpoint

```
GET https://api.verifik.co/v3/ec/company
```

Interrogez une société équatorienne avec **`documentType`** `RUCEC` et **`documentNumber`** (13 chiffres, sans séparateurs). Retourne des champs orientés enregistrement — notamment l'activité économique, le régime et les représentants légaux — pour soutenir le **KYB**, le **contrôle fournisseurs** et la **conformité**.

### En-têtes (Headers)

| Nom            | Valeur              |
| -------------- | ------------------- |
| Content-Type   | `application/json`  |
| Authorization  | `Bearer <token>`    |

### Paramètres

| Nom              | Type   | Requis  | Description                                              | Exemple         |
| ---------------- | ------ | ------- | -------------------------------------------------------- | --------------- |
| `documentType`   | string | **Oui** | Type de document. Valeur autorisée : `RUCEC`.            | `RUCEC`         |
| `documentNumber` | string | **Oui** | RUCEC à 13 chiffres, sans espaces ni points.             | `1793197955001` |

### Notes

- `documentNumber` doit comporter exactement 13 chiffres, sans séparateurs.
- RUC de démonstration sandbox : `1790008959001` (ACERO COMERCIAL ECUATORIANO S.A.).
- Exemples live : `1793197955001`, `1191795453001`.
- L'endpoint legacy `GET /v2/ec/company` reste disponible avec une forme de réponse plus légère.
