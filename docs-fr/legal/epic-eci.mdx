# Inde — Recherche de nom EPIC (Voter ID)

Verifik expose un endpoint REST unique pour consulter les **informations de nom** liées à un **EPIC** indien (*Elector's Photo Identity Card*) à partir de sources alignées sur la **Commission électorale de l'Inde (ECI)**. Utilisez-le pour les contrôles d'identité, l'onboarding et les flux de conformité qui doivent confirmer qu'un numéro de Voter ID renvoie des données de nom cohérentes.

## Que valide cette API ?

L'API se connecte aux données électorales officielles pour renvoyer :

- **Structure du nom de l'électeur** : prénom et nom (y compris les variantes en écriture locale si disponibles).
- **Nom complet et nom du parent** : champs adaptés à l'affichage et à la comparaison en alphabet latin et en écriture locale.
- **Correspondance EPIC** : confirme que l'EPIC de **10 caractères** (lettres et chiffres) est reconnu et renvoie les enregistrements associés.

Cela réduit les contrôles manuels et maintient la vérification électorale alignée sur le registre sous-jacent.

## Référence de l'API

### Endpoint

```
https://api.verifik.co/v2/in/epic
```

Envoyez une requête **GET** avec l'EPIC dans le paramètre de requête `documentNumber`. Le service renvoie des champs de nom structurés, des valeurs optionnelles en écriture locale, et un bloc **signature** Verifik en cas de succès.

### En-têtes (Headers)

| Nom           | Valeur             |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### Paramètres

| Nom              | Type   | Requis   | Description                                                                   | Exemple     |
| ---------------- | ------ | -------- | ----------------------------------------------------------------------------- | ----------- |
| `documentNumber` | string | **Oui**  | EPIC / Voter ID. Doit faire **exactement 10 caractères** (alphanumérique). | `IHM2796746` |

### Notes

- `documentNumber` doit faire **exactement 10 caractères** ; une longueur ou un format invalide peut renvoyer `409`.
- Les données sont obtenues **via Verifik** ; la couverture suit la disponibilité ECI pour l'EPIC donné.
- Pour les détails de **bureau de vote et circonscription**, utilisez l'endpoint compagnon [**Inde — Informations de vote / bureau EPIC**](/verifik-fr/legal/voting-eci-epic) (`GET /v2/in/epic/votacion`).

## Cas d'usage courants

- **Programmes KYC et d'identité** qui doivent corroborer un Voter ID avec les champs de nom et de parenté renvoyés.
- **Services publics et citoyens** qui doivent vérifier un EPIC avant de délivrer des accès ou des identifiants.
- **Fintech et conformité** où les contrôles d'identité électorale font partie d'une stratégie plus large anti-fraude.

## Sources officielles et fiabilité

Verifik achemine les requêtes vers des sources électorales de confiance afin que vous receviez des données cohérentes avec le comportement de recherche publié par l'ECI. Tous les appels utilisent TLS en transit ; traitez les EPIC comme des **données personnelles sensibles** dans vos politiques de journalisation et de stockage.

## Avantages clés

- **Intégration simple** : un seul GET avec `documentNumber` et un en-tête `Authorization` standard.
- **Réponse structurée** : tableaux de noms et champs en écriture locale lorsque la source les fournit.
- **Certification Verifik** : chaque réponse réussie peut inclure un bloc `signature` Verifik pour l'audit.

## Conformité et sécurité

Stockez les jetons de façon sécurisée, faites tourner les clés API, et respectez les règles de protection des données de votre juridiction pour les données électorales et d'identité.
