# Chili — Consultation contribuable (RUT)

Utilisez cet endpoint lorsque vous avez besoin de données d'identité **orientées contribuable** pour un **RUT** chilien (et non du flux registre civil par cedula). Il complète **[Vérification d'identité au Chili](/identity/chile)** (`v2/cl/cedula`), qui est ajusté pour la consultation **RUN/civil**.

## Référence de l'API

### Endpoint

```
https://api.verifik.co/v2/cl/taxpayer
```

Retourne les champs de nom et identifiants alignés avec l'usage **registre fiscal** — idéal pour le **KYB**, l'**onboarding fournisseurs** et la **validation de factures** lorsque le **RUT** doit correspondre aux données du **contribuable enregistré**.

### En-têtes (Headers)

| Nom | Valeur |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Paramètres

| Nom | Type | Requis | Description |
| --- | --- | --- | --- |
| `documentType` | string | **Oui** | Doit valoir `RUT`. |
| `documentNumber` | string | **Oui** | Identifiant fiscal (chiffres ; le formatage est normalisé par le service). |

### Requête (Request)

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/cl/taxpayer", {

  headers: { Accept: "application/json", Authorization: `Bearer ${process.env.VERIFIK_TOKEN}` },
});
console.log(data);
```

### Notes

- Privilégiez **`v2/cl/cedula`** pour la correspondance d'identité **RUN** au registre civil ; utilisez **`v2/cl/taxpayer`** lorsque le dossier **contribuable RUT** fait foi.
