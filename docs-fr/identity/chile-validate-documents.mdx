# Chili — Validation de document d'identité

Cet endpoint contrôle la **vigence** du document pour un **RUN** ou **RUT** chilien en utilisant le **numéro de série** imprimé sur la pièce, en plus de l'identifiant. Servez-vous-en lorsque vous devez confirmer que la **carte** est **valide** (pas seulement que le numéro existe)—par exemple pour un **KYC à assurance élevée** ou des exigences de **conformité** au-delà d'une simple consultation de cédula.

Pour une vérification nominale au registre civil **sans** série, utilisez la **[Vérification d'identité au Chili](/verifik-fr/identity/chile)** (`v2/cl/cedula`).

## API Reference

### Endpoint

```
https://api.verifik.co/v2/cl/validate/documents
```

Soumettez **RUN** ou **RUT**, **numéro de document** et **série imprimée** pour obtenir un **statut de vigence** depuis les sources civiles ou fiscales—à utiliser lorsque le document physique doit être confirmé, pas seulement le numéro.

### Headers

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Parameters

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `documentType` | string | Yes | `RUN` (flux civil) ou `RUT` (flux fiscal). |
| `documentNumber` | string | Yes | Identifiant sans espaces, selon le guide d'intégration. |
| `serialNumber` | string | Yes | Série imprimée ; requise par les sources sous-jacentes. |

### Request

```javascript
import axios from "axios";
```

### Notes

- La chaîne exacte de `status` dépend du registre amont ; traitez-la comme **indicateur de vigence**, pas comme conseil juridique.
- Collectez **`serialNumber`** sur le document physique ; sans elle, la validation ne peut pas se compléter via le flux **Registro Civil**.
