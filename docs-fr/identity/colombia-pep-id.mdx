### Point d'accès

```
GET https://api.verifik.co/v2/co/foreigner-id/pep
```

Envoyez **`documentNumber`** et **`expeditionDate`** (`DD/MM/YYYY`) en **query**. Même schéma en **`POST`** (JSON).

### En-têtes

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Exigences documentaires

**Pour qui ?** **Ressortissants vénézuéliens** en Colombie titulaires d'un *Permiso Especial de Permanencia* (**PEP**) — **permis d'immigration**, pas le dépistage LCB « personne politiquement exposée » (voir [PEP colombien (LCB)](/verifik-fr/identity-validation/colombia/colombian-politically-exposed-persons)).

| Champ | Consignes |
| --- | --- |
| **`documentNumber`** | Numéro PEP, **chiffres uniquement**. En Colombie, ce numéro fait **toujours 15 chiffres**. Envoyez les 15 chiffres sans espaces ni ponctuation. Exemple : `123456789012345`. |
| **`expeditionDate`** | Date d'émission au format **`JJ/MM/AAAA`** (ex. `10/10/2024`). Obligatoire. |

Comparaison complète : [Guide des documents d'identité](/verifik-fr/identity-validation/colombia/colombia-identity-documents-guide).

### Paramètres

| name | type | required | description |
| --- | --- | --- | --- |
| `documentNumber` | string | yes | Numéro PEP, **chiffres uniquement**. **15 chiffres** (longueur fixe en Colombie). Exemple : `123456789012345`. |
| `expeditionDate` | string | yes | Date d'émission au format **`JJ/MM/AAAA`** (ex. `10/10/2024`). Doit correspondre au document. |

### Requête

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/co/foreigner-id/pep", {
	params: { documentNumber: "123456789", expeditionDate: "10/10/2024" },
	headers: {
		Accept: "application/json",
		Authorization: `Bearer ${process.env.VERIFIK_TOKEN}`,
	},
});
console.log(data);
```
