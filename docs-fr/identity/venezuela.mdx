# Vérification d'identité au Venezuela

L'API de vérification d'identité de Verifik vous permet d'authentifier les citoyens vénézuéliens à partir des données officielles du gouvernement. Elle est conçue pour fluidifier vos processus KYC (Know Your Customer), prévenir la fraude et vous assurer de respecter sans effort toutes les exigences réglementaires.

Nous avons construit cette intégration pour les entreprises qui ont besoin d'un moyen rapide, sécurisé et automatisé de confirmer la véritable identité de leurs utilisateurs, employés ou clients.

## Que valide cette API ?

Notre API se connecte directement aux registres officiels pour valider :

-   **Nom complet et numéro de document** : prend en charge les pièces d'identité vénézuéliennes (CCVE).
-   **Correspondance d'identité** : confirme que le nom fourni correspond au numéro de document.
-   **Données personnelles facultatives** : peuvent inclure la date de naissance, le sexe et l'état civil lorsqu'elles sont disponibles.

En vérifiant ces éléments, vous pouvez avoir la certitude que la personne avec qui vous traitez existe et possède un document valide, ce qui réduit considérablement les risques d'usurpation d'identité et de fraude.

## Référence de l'API

### Endpoint

```
https://api.verifik.co/v2/ve/cedula
```

Cette route retourne les données d'identité des **ressortissants vénézuéliens** via la classe de document **CCVE** (*Cédula de Identidad*). Transmettez le numéro de cédula via **`documentNumber`** ; l'API résout l'enregistrement et retourne des champs de nom structurés pour le **KYC**, l'onboarding et les contrôles de fraude. Pour les **résidents étrangers**, utilisez plutôt [`GET /v2/ve/foreigner-id`](/identity/venezuela-foreigner-id) (CEVE).

### En-têtes (Headers)

| Nom            | Valeur              |
| -------------- | ------------------- |
| Accept         | `application/json`  |
| Authorization  | `Bearer <token>`    |

### Paramètres

| Nom              | Type   | Requis  | Description                                                                                                  |
| ---------------- | ------ | ------- | ------------------------------------------------------------------------------------------------------------ |
| `documentNumber` | string | **Oui** | Numéro de cédula nationale (CCVE). Aucun paramètre `documentType` n'est requis — le service applique **CCVE** pour cet endpoint. |

### Notes

- `documentNumber` est requis.
- Certains champs peuvent figurer dans la réponse en fonction de la disponibilité des données : `birthDate`, `gender` et `maritalStatus`. Leur présence n'est pas garantie pour toutes les réponses.
- Lorsqu'il est présent, `birthDate` suit le format `AAAA-MM-JJ` (par ex. "1998-03-01").
