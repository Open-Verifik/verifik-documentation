# Chile — Informações do veículo

O serviço de validação veicular no Chile permite obter informações detalhadas sobre veículos registrados no país. Use para **frotas**, **seguros**, **garantias** e conformidade operacional.

## Consulta por placa

### Endpoint

```
https://api.verifik.co/v2/cl/vehicle
```

Consulte pela **placa** (`plate`, sem espaços ou pontos). Em geral são retornados **marca**, **modelo**, **ano**, **chassi/VIN**, **motor**, **cor**, **tipo**, **resumo de multas** e **RUT do proprietário** quando o registro expõe esses dados.

### Headers

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Parameters

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `plate` | string | Yes | Placa sem espaços ou pontos. |

### Request

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/cl/vehicle", {
  params: { plate: "BB985" },
  headers: { Accept: "application/json", Authorization: `Bearer ${process.env.VERIFIK_TOKEN}` },
});
console.log(data);
```

  </TabItem>
  <TabItem value="python" label="Python">

```python
import os, requests

url = "https://api.verifik.co/v2/cl/vehicle"
headers = {"Accept": "application/json", "Authorization": f"Bearer {os.getenv('VERIFIK_TOKEN')}"}
params = {"plate": "BB985"}
r = requests.get(url, headers=headers, params=params)
print(r.json())
```

## Recursos

### Consulta de informações
- **Cadastro**: verificação de registro no Chile.
- **Especificações**: chassi, motor, fabricante, modelo.
- **Detalhes**: ano, cor, tipo.
- **Proprietário**: RUT quando disponível.
- **Multas**: situação de débitos/multas associados.
- **Roubo/furto**: quando disponível na fonte.
- **Transporte público**: classificação quando aplicável.

### Geral
- **Vários exemplos de código**: Node.js, Python e integrações comuns.
- **Dados em tempo real**: conforme a fonte oficial.
- **Erros estruturados**: códigos HTTP e mensagens claras.
- **JSON assinado**: campo `signature` quando aplicável.
