# 🇧🇷 Brasil — Informações do veículo

## Consulta de informações do veículo

### Endpoint

```
GET https://api.verifik.co/v2/br/vehicle
```

Com uma **placa** brasileira, a **Verifik** devolve em uma única requisição dados estruturados do veículo para vincular um ativo físico a atributos verificados e normalizados. É voltada a **onboarding de frota e mobilidade**, **cotação e sinistros de seguro**, **financiamento garantido e análise de colateral**, e **checagens em marketplace ou concessionária** quando a placa é o identificador principal.

A API Verifik devolve um **registro normalizado do veículo** para essa placa, incluindo **placa**, **marca**, **modelo**, **ano modelo** e **ano de fabricação**, **chassi (VIN)**, **motor**, **cor**, **tipo de carroceria**, **combustível**, **transmissão**, **portas**, **fabricante ou planta**, **códigos FIPE** quando se aplicam ao veículo, e **indicadores de irregularidades ou restrições** quando existem para aquele cadastro. Use para confirmar se a unidade corresponce à linha oficial antes de seguir.

### Cabeçalhos

| Nome          | Valor              |
| ------------- | ------------------ |
| Accept        | `application/json` |
| Authorization | `Bearer <token>`   |

### Parâmetros

| Nome    | Tipo   | Obrigatório | Descrição |
| ------- | ------ | ----------- | --------- |
| `plate` | string | **Sim**     | Placa a consultar (letras e números; a capitalização é normalizada). Exemplo: `PZY7H82`. |

### Campos de resposta

| Campo | Tipo | Descrição |
| ----- | ---- | --------- |
| `bodyType` | string | Tipo de veículo retornado pelo registro. |
| `brand` | string | Marca do veículo. |
| `chassis` | string | Chassi ou VIN retornado pelo registro. |
| `color` | string | Cor registrada do veículo. |
| `country` | string | País do registro do veículo. |
| `denatranWarning` | string | Status ou aviso do Denatran, como `Sem Restrição`. |
| `doors` | string | Capacidade de passageiros retornada pelo registro. |
| `engine` | string | Cilindrada ou referência de motor no registro. |
| `factory` | string | Origem ou região de fabricação. |
| `fipeCodes` | string[] | Códigos FIPE relacionados ao veículo quando disponíveis. |
| `fuelType` | string | Tipo de combustível registrado. |
| `irregularitiesCount` | number | Número de restrições ou irregularidades ativas encontradas. |
| `irregularityCode` | string | Restrições ativas unidas em uma string; vazio quando não há restrições. |
| `manufacturer` | string | Fabricante retornado pelo registro. |
| `model` | string | Descrição do modelo registrado. |
| `modelYear` | string | Ano do modelo. |
| `plate` | string | Placa normalizada retornada pela consulta. |
| `transmission` | string | Transmissão quando disponível; registros obtidos pela fonte registral podem retornar uma string vazia. |
| `vehicle` | string | Categoria do veículo no registro. |
| `version` | string | Melhor correspondência de modelo/versão FIPE quando disponível. |
| `yearOfManufacture` | string | Ano de fabricação. |

### Notas

- Apenas **`plate`** é obrigatório na query string.
- Registros obtidos pela fonte registral podem retornar **`transmission`** como uma string vazia.
- **`doors`** reflete a capacidade de passageiros da fonte registral.
- **`fipeCodes`** pode ser um array vazio quando não houver correspondência FIPE.
- Provedores de fallback podem retornar cobertura de campos ligeiramente diferente quando a fonte primária não encontrar dados.
- Algumas contas também habilitam o produto **ficha técnica estendida por placa**; use a URL do seu catálogo AppFeature quando esse recurso estiver ativo.
