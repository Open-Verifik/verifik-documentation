# 🇦🇷 Argentina — RTO

### Endpoint

```
GET https://api.verifik.co/v2/ar/rto
```

Use este endpoint para consultar registros de inspeção RTO argentina pela placa. A resposta inclui resultado da inspeção, tipo de inspeção, datas de inspeção e vencimento, certificado, categoria de escalabilidade, centro de inspeção e tipo de alerta.

### Cabeçalhos

| Name          | Value              |
| ------------- | ------------------ |
| Accept        | `application/json` |
| Authorization | `Bearer <token>`   |

### Parâmetros

| Nome | Tipo | Obrigatório | Descrição |
| ---- | ---- | -------- | ----------- |
| `plate` | string | **Yes** | Formato Mercosur `AA-123-BB` (envie `AA123BB`, sem espaços ou pontuação). |

### Campos de resposta

| Campo | Tipo | Descrição |
| ----- | ---- | ----------- |
| `plate` | string | Placa normalizada. |
| `totalRecords` | string | Número de registros RTO retornados. |
| `inspections` | object[] | Registros de inspeção RTO. |
| `inspections[].result` | string \| null | Resultado da inspeção. |
| `inspections[].inspectionType` | string \| null | Rótulo do tipo de inspeção. |
| `inspections[].inspectionDate` | string \| null | Data da inspeção. |
| `inspections[].expirationDate` | string \| null | Data de vencimento. |
| `inspections[].certificate` | string \| null | Identificador do certificado. |
| `inspections[].scalabilityCategory` | string \| null | Categoria de escalabilidade retornada pela fonte. |
| `inspections[].inspectionCenter` | string \| null | Nome do centro de inspeção. |
| `inspections[].alertType` | string \| null | Tipo de alerta da fonte quando disponível. |

### Notas

- Campos de data são normalizados quando a fonte retorna um valor dia/mês/ano reconhecível.
- Registros RTO são retornados como array `inspections` para lidar com múltiplos registros por placa.
