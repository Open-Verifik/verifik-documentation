# 🇨🇱 Chile — SOAP veicular (por placa)

Consulte o **SOAP** veicular (*Seguro Obligatorio de Accidentes Personales*) no Chile usando **somente a placa**. O serviço consulta Boostr, enriquece com AACH quando há número de apólice e retorna atributos do veículo via Carvuk quando o SOAP está ausente.

Payload **aninhado** quando a fonte responde:

- **`plate`** — Placa chilena (Patente), **4–6** caracteres alfanuméricos
- **`message`** — vazio se SOAP encontrado; `No_active_SOAP_certificate_found_for_this_plate` se o veículo existe sem SOAP ativo
- **`soap`** — se encontrado: **`status`**, **`policyNumber`**, **`insuranceCompany`**, **`validFrom`**, **`validTo`**, **`folioNumber`**, **`ownerName`**, **`ownerRut`**, **`premium`**
- **`vehicle`** — **`brand`**, **`model`**, **`year`**, **`type`**, **`engineNumber`**
- Resposta **assinada** Verifik

Use para **conformidade de seguros**, **onboarding de frotas** e **sinistros** quando só a placa é conhecida.

### Endpoint

```
GET v2/cl/vehicle-soap-plate
```

Consulte por **placa** (`plate`) apenas. Campos típicos em **`soap`**: **status**, **insuranceCompany**, **ownerName**, **validFrom**, **validTo**; em **`vehicle`**: **brand**, **model**, **year**.

### Parâmetros

### Notas

- **Sem SOAP ativo** retorna **HTTP 200** com **`soap` vazio** e **`message: No_active_SOAP_certificate_found_for_this_plate`** — não é 404.
- **Placas live:** `CLTK27`, `HTRT86`, `RRWX51`, `THHV64`, `CSXZ52`, `CSXZ57` retornam SOAP ativo; `CSXZ27` retorna veículo sem SOAP ativo.
- **Modo sandbox:** **`AB1001`**–**`AB1010`** e **`HTRT86`** retornam perfis aninhados estáveis; **`ERR404`** retorna **404**.
- Para placa + apólice, veja [SOAP veicular (placa + apólice)](/verifik-pt/vehicle-validation/chile/vehicle-soap).
- A **disponibilidade** do registro depende do ambiente; indisponibilidade pode retornar **409** (timeout).
