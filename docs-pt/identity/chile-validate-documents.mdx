# Chile — Validação de documento de identidade

Este endpoint verifica a **vigência** do documento para **RUN** ou **RUT** chileno usando o **número de série** impresso na cédula ou cartão, além do identificador. Use quando precisar confirmar que o **documento físico** é **válido**, não só que o número existe—por exemplo em **KYC de alta garantia** ou etapas de **compliance** além de uma consulta simples à cédula.

Para verificação apenas nominal no registro civil **sem** série, use a **[Verificação de identidade no Chile](/identity/chile)** (`v2/cl/cedula`).

## API Reference

### Endpoint

```
https://api.verifik.co/v2/cl/validate/documents
```

Envie **RUN** ou **RUT**, **número do documento** e **série impressa** para obter um **status de vigência** a partir de fontes civis ou tributárias—use quando o documento físico precisa ser confirmado, não apenas o número.

### Headers

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Parameters

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `documentType` | string | Yes | `RUN` (fluxo civil) ou `RUT` (fluxo tributário). |
| `documentNumber` | string | Yes | Identificador sem espaços, conforme o guia de integração. |
| `serialNumber` | string | Yes | Série impressa no documento; exigida pelas fontes de validação. |

### Request

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/cl/validate/documents", {
  params: {
    documentType: "RUN",
    documentNumber: "18562530-3",
    serialNumber: "A123456789",
  },
  headers: { Accept: "application/json", Authorization: `Bearer ${process.env.VERIFIK_TOKEN}` },
});
console.log(data);
```

### Notes

- O texto exato de `status` depende do registro de origem; trate como **indicador de vigência**, não como parecer jurídico.
- Colete **`serialNumber`** do documento físico; sem ele, a validação pelo fluxo do **Registro Civil** não completa.
