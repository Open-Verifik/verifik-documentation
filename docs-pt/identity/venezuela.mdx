# Verificação de identidade na Venezuela

A API de verificação de identidade da Verifik permite autenticar cidadãos venezuelanos usando dados oficiais do governo. Foi projetada para automatizar seus processos de KYC (*Know Your Customer*), prevenir fraudes e garantir o cumprimento de todos os requisitos regulatórios sem esforço.

Construímos esta integração para empresas que precisam de uma forma rápida, segura e automatizada de confirmar a verdadeira identidade de usuários, funcionários ou clientes.

## O que esta API valida?

Nossa API conecta-se diretamente aos registros oficiais para validar:

-   **Nome completo e número do documento**: suporta documentos de identidade venezuelanos (CCVE).
-   **Correspondência de identidade**: confirma que o nome informado corresponde ao número do documento.
-   **Detalhes pessoais opcionais**: pode incluir data de nascimento, gênero e estado civil quando disponíveis.

Ao validar esses elementos, você pode confiar que a pessoa com quem está negociando é real e possui um documento válido, reduzindo significativamente os riscos de personificação e fraude.

## Referência da API

### Endpoint

```
https://api.verifik.co/v2/ve/cedula
```

Esta rota retorna dados de identidade para **cidadãos venezuelanos** usando a classe de documento **CCVE** (*Cédula de Identidad*). Envie o número da cédula como **`documentNumber`**; a API resolve o registro e retorna campos de nome estruturados para **KYC**, onboarding e verificações de fraude. Para **residentes estrangeiros**, use [`GET /v2/ve/foreigner-id`](/identity/venezuela-foreigner-id) (CEVE).

### Cabeçalhos

| Nome          | Valor              |
| ------------- | ------------------ |
| Accept        | `application/json` |
| Authorization | `Bearer <token>`   |

### Parâmetros

| Nome           | Tipo    | Obrigatório | Descrição                                    |
| -------------- | ------- | ----------- | -------------------------------------------- |
| `documentNumber` | string | Sim        | Número da cédula nacional (CCVE). Não é necessário enviar `documentType` — o serviço aplica **CCVE** para este endpoint. |

### Observações

- `documentNumber` é obrigatório.
- Alguns campos podem ser incluídos na resposta dependendo da disponibilidade dos dados: `birthDate`, `gender` e `maritalStatus`. Esses campos não são garantidos em todas as respostas.
- Quando presente, `birthDate` segue o formato `AAAA-MM-DD` (ex.: "1998-03-01").
