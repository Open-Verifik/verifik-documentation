### Endpoint

```
GET https://api.verifik.co/v2/co/foreigner-id/ppt
```

**`documentNumber`** e **`expeditionDate`** (`DD/MM/YYYY`). **`POST`** com o mesmo JSON.

### Cabeçalhos

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Requisitos do documento

**Para quem?** **Nacionais venezuelanos** na Colômbia com *Permiso de Protección Temporal* (**PPT**) quando você precisa do **status migratório** da **Migración Colombia** (ex. VIGENTE, datas de vencimento).

| Campo | Orientação |
| --- | --- |
| **`documentNumber`** | Número PPT, **somente dígitos**. Na Colômbia o permiso costuma ter **até 7 dígitos**. Alguns sistemas (SENA, folha) exibem preenchido até **15 caracteres** — envie o valor numérico dos seus registros. Exemplo Migración: `2081381`. |
| **`expeditionDate`** | Data de expedição **`DD/MM/AAAA`** (ex. `07/03/2022`). Obrigatória. |

**Também disponível:** consulta de nome via [`/v2/co/cedula`](/verifik-pt/identity-validation/colombia/colombia-identity-documents-guide) com `documentType=PPT` (validação 5–10 dígitos, sem data de expedição).

Comparação completa: [Guia de documentos de identidade](/verifik-pt/identity-validation/colombia/colombia-identity-documents-guide).

### Parâmetros

| name | type | required | description |
| --- | --- | --- | --- |
| `documentNumber` | string | yes | Número PPT, **somente dígitos**. Geralmente **até 7 dígitos** (alguns sistemas preenchem até 15). Exemplo: `2081381`. |
| `expeditionDate` | string | yes | Data de expedição **`DD/MM/AAAA`** (ex. `07/03/2022`). Deve coincidir com o documento. |

### Requisição

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/co/foreigner-id/ppt", {
	params: { documentNumber: "123456789", expeditionDate: "07/03/2022" },
	headers: {
		Accept: "application/json",
		Authorization: `Bearer ${process.env.VERIFIK_TOKEN}`,
	},
});
console.log(data);
```

### Notas

- Data em **`DD/MM/YYYY`**.
- **`GET`** e **`POST`** equivalentes.
- Falhas no portal podem gerar **404** / **500**.
