# Índia — Consulta de nome EPIC (Voter ID)

A Verifik expõe um único endpoint REST para consultar **informações de nome** vinculadas a um **EPIC** indiano (*Elector's Photo Identity Card*) a partir de fontes alinhadas com a **Comissão Eleitoral da Índia (ECI)**. Use-o em checagens de identidade, onboarding e fluxos de conformidade que precisem confirmar que um Voter ID retorna dados de nome consistentes.

## O que esta API valida?

A API conecta-se a dados eleitorais oficiais para retornar:

- **Estrutura do nome do eleitor**: nome e sobrenome (incluindo variantes em escrita local quando disponíveis).
- **Nome completo e nome do parente**: campos adequados para exibição e comparação em alfabeto latino e escrita local.
- **Correspondência do EPIC**: confirma que o EPIC de **10 caracteres** (letras e números) é reconhecido e retorna os registros associados.

Isso reduz checagens manuais e mantém a verificação eleitoral alinhada ao registro subjacente.

## Referência da API

### Endpoint

```
https://api.verifik.co/v2/in/epic
```

Envie uma requisição **GET** com o EPIC no parâmetro de query `documentNumber`. O serviço retorna campos de nome estruturados, valores opcionais em escrita local e um bloco **signature** da Verifik em caso de sucesso.

### Headers

| Nome          | Valor              |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### Parâmetros

| Nome             | Tipo   | Obrigatório | Descrição                                                                     | Exemplo     |
| ---------------- | ------ | ----------- | ----------------------------------------------------------------------------- | ----------- |
| `documentNumber` | string | **Sim**     | EPIC / Voter ID. Deve ter **exatamente 10 caracteres** (alfanumérico).      | `IHM2796746` |

### Notas

- `documentNumber` deve ter **exatamente 10 caracteres**; comprimento ou formato inválido pode retornar `409`.
- Os dados são obtidos **pela Verifik**; a cobertura segue a disponibilidade da ECI para o EPIC informado.
- Para detalhes de **local de votação e circunscrição**, use o endpoint complementar [**Índia — Informações de votação / local EPIC**](/verifik-pt/legal/voting-eci-epic) (`GET /v2/in/epic/votacion`).

## Casos de uso comuns

- **Programas de KYC e identidade** que precisam corroborar um Voter ID com nome e nome do parente retornados.
- **Serviços públicos e cidadãos** que devem verificar um EPIC antes de emitir credenciais ou acesso.
- **Fintech e compliance** em que checagens de identidade eleitoral fazem parte de uma estratégia maior de risco e fraude.

## Fontes oficiais e confiabilidade

A Verifik encaminha as requisições a fontes eleitorais confiáveis para que você receba dados consistentes com o comportamento de busca publicado pela ECI. Todas as chamadas usam TLS em trânsito; trate EPICs como **dados pessoais sensíveis** nas suas políticas de log e armazenamento.

## Principais benefícios

- **Integração simples**: um GET com `documentNumber` e um header `Authorization` padrão.
- **Resposta estruturada**: arrays de nome e campos em escrita local quando a fonte os fornece.
- **Certificação Verifik**: cada resposta bem-sucedida pode incluir um bloco `signature` da Verifik para auditoria.

## Conformidade e segurança

Use apenas HTTPS (a URL acima é protegida por TLS). Armazene tokens com segurança, rotacione chaves de API e siga as regras de proteção de dados da sua jurisdição para dados eleitorais e de identidade. A plataforma Verifik é projetada para alta disponibilidade e acesso monitorado.
