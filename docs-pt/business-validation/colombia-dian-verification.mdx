# 🇨🇴 Colômbia — Verificação DIAN

A API de Verificação DIAN da Verifik ajuda a autenticar o status fiscal de empresas colombianas usando dados governamentais oficiais. Foi projetada para automatizar seus processos KYB (*Know Your Business*), prevenir fraudes e garantir o cumprimento de todos os requisitos regulatórios sem esforço.

Construímos esta integração para empresas que precisam de uma forma rápida, segura e automatizada de confirmar o status fiscal e a legitimidade de empresas, parceiros ou fornecedores colombianos.

## O que esta API valida?

Nossa API conecta-se diretamente aos registros oficiais da DIAN (*Dirección de Impuestos y Aduanas Nacionales*) para validar:

-   **Razão social e NIT**: suporta o *Número de Identificación Tributaria* (NIT).
-   **Status fiscal**: retorna o status de registro da empresa na DIAN.
-   **Correspondência da empresa**: confirma que as informações correspondem ao NIT e valida responsabilidades fiscais ativas.

Ao validar esses elementos, você pode confiar que a empresa com quem está negociando é legítima, está corretamente registrada na autoridade fiscal e possui obrigações fiscais ativas, reduzindo significativamente o risco de fraude e garantindo conformidade.

## Referência da API

### Ponto de acesso

```
GET https://api.verifik.co/v2/co/company/dian
```

Mesmo caminho no app: `GET https://verifik.app/v2/co/company/dian`.

### Cabeçalhos

| Nome | Valor |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Parâmetros

| Nome | Tipo | Obrigatório | Descrição |
| --- | --- | --- | --- |
| `documentType` | string | Sim | Deve ser **`NIT`**. |
| `documentNumber` | string | Sim | NIT. Espaços são removidos no servidor; você pode incluir ou omitir o dígito verificador. Comprimento mínimo **5** caracteres (validação da API). |

### Notas

- **`GET` apenas em query** — sem corpo. Use **`Accept`** (não `Content-Type`) para respostas JSON.
- Use uma string NIT válida; a integração trata formatação e dígito verificador conforme retornado pela DIAN.
- **Divisão de nome para pessoas naturais**: quando a resposta é de uma « persona natural », a Verifik divide automaticamente o nome do campo `nombreRazon` em `firstName`, `lastName`, `fullName` e `arrayName`. Como a fonte retorna o nome em um único campo (`nombreRazon`), a divisão pode não ser 100% precisa. O campo `nombreRazon` sempre contém o nome original retornado pela DIAN.
