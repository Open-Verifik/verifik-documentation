コロンビア — 国民身分証 プレミアム (CC)

目的: コロンビア CC（国民身分証） 番号を公的ソースに照合し、KYC および法令対応向けの構造化本人情報を返します。戻り値は名義、生年月日、発行日時・発行地、ソースが返す範囲で性別・生存等を含み、署名付きの認証ブロックを含みます。送信は番号のみ—発行日はサーバ側で解決（リクエストに文書区分・発行日 なし）。属性の粒度は cédula extra と同相当です。消費クレジットは基本 /v2/co/cedula より高く、連鎖的な前処理分が含まれます。

API リファレンス​

エンドポイント​

GET https://api.verifik.co/v2/co/cedula/premium

レスポンス: data（当該番号に対しソースが返す本人属性）、signature（ペイロードの認証メタデータ）、id（要求識別子）を返します。HTTP 404 は、本フローでは当該番号について完全な本人レコードを組み立てられなかったことを示します。HTTP 409 は、上流連携前の入力妥当性検証に失敗（例: documentNumber の長さ）を示します。クレジットは基本 /v2/co/cedula より高（解決の連鎖のため）。CC のみ、通常 5～10 桁の数字。

ヘッダー​

名前値
Acceptapplication/json
AuthorizationBearer <token>

パラメータ​

名前型必須説明
documentNumberstringはいコロンビア CC（Cédula de Ciudadanía）の番号。正規化で非数字を除去し、5～10 桁の数字として検証します。

動的料金​

標準エンドポイント GET/POST /v2/co/cedula（基本 cédula）は Verifik の動的クエリアーキテクチャに含まれます。通常は標準レートが請求されます。標準検証パスが一致を返さない場合、拡張検証パスが自動実行されることがあります。そのパスが HTTP 200 を返すと動的料金が適用され、クレジットは標準ティアではなくこのエンドポイントファミリーのプレミアムティアで差し引かれます。

料金の目安： アカウントの標準レートからプレミアムレートまで（プランまたは Postman を参照）。

/v2/co/cedula で includeCost=true を指定すると、動的料金が適用された場合に billing オブジェクトを受け取れます。SLA — 動的料金（請求）を参照してください。

直接プレミアム vs 動的料金​

このページは明示的なプレミアムエンドポイント（/v2/co/cedula/premium）を説明しています。直接呼び出すと常にプレミアム料金が適用されます。

動的料金は /v2/co/cedula からの自動エスカレーションを指し、成功時（HTTP 200）はこのルートと同じプレミアムティアが請求されます。

リクエスト例​

  • GET
  • POST
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/co/cedula/premium", {
	params: { documentNumber: "1234567890" },
	headers: {
		Accept: "application/json",
		Authorization: `Bearer ${process.env.VERIFIK_TOKEN}`,
	},
});
console.log(data);
import axios from "axios";

const { data } = await axios.post(
	"https://api.verifik.co/v2/co/cedula/premium",
	{ documentNumber: "1234567890" },
	{
		headers: {
			Accept: "application/json",
			Authorization: `Bearer ${process.env.VERIFIK_TOKEN}`,
		},
	}
);
console.log(data);

レスポンス​

  • 200
  • 404
  • 409

サンプル形状（プレースホルダ。実在人物ではありません）：

{
	"data": {
		"arrayName": ["GIVEN", "MIDDLE", "PATERNAL", "MATERNAL"],
		"dateOfBirth": "1990-05-20",
		"documentNumber": "1234567890",
		"documentType": "CC",
		"expeditionDate": "2015-08-12",
		"expeditionPlace": {
			"municipio": "サンプル市",
			"departamento": "サンプル州"
		},
		"firstName": "GIVEN MIDDLE",
		"fullName": "GIVEN MIDDLE PATERNAL MATERNAL",
		"gender": "HOMBRE",
		"isAlive": true,
		"lastName": "PATERNAL MATERNAL"
	},
	"signature": {
		"dateTime": "April 21, 2026 9:34 PM",
		"message": "Certified by Verifik.co"
	},
	"id": "XXXXX"
}

実フィールドの有無は公式ソースの状況に依存します。

{
	"code": "NotFound",
	"message": "Record not found."
}

上流が一致または必要データを返せない場合（例: 発行日解決の失敗）。

{
	"code": "MissingParameter",
	"message": "documentNumber maximum length: 10\n"
}

documentNumber の長さは 5～10 桁。入力不備時は他の Joi メッセージが出ることもあります。
