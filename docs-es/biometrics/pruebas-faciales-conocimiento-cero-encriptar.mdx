### Endpoint

```
POST https://api.verifik.co/v2/zelf-proof/encrypt
```

Crea una prueba facial de conocimiento cero encriptando datos faciales y almacenándolos en IPFS. La prueba puede ser verificada posteriormente sin exponer los datos biométricos originales.

### Headers

| Name | Value |
| --- | --- |
| Content-Type | `application/json` |
| Authorization | `Bearer <token>` |

### Parámetros

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `faceBase64` | string | Yes | Imagen facial codificada en Base64 |
| `requireLiveness` | boolean | Yes | Si requiere detección de vitalidad |
| `livenessLevel` | string | Yes | Nivel de detección de vitalidad (`REGULAR`, `SOFT`, `HARDENED`) |
| `os` | string | Yes | Sistema operativo (`DESKTOP`, `ANDROID`, `IOS`) |
| `identifier` | string | Yes | Identificador único (solo alfanumérico, sin espacios ni caracteres especiales) |
| `publicData` | object | Yes | Datos públicos a almacenar con la prueba (pares clave-valor de string) |
| `metadata` | object | Yes | Metadatos privados (pares clave-valor de string) |
| `password` | string | No | Contraseña opcional para seguridad adicional |
| `referenceFaceBase64` | string | No | Cara de referencia para comparación |
| `tolerance` | string | No | Nivel de tolerancia (`REGULAR`, `SOFT`, `HARDENED`) |
| `verifierKey` | string | No | Clave de verificador para validación adicional |
| `livenessDetectionPriorCreation` | boolean | No | Habilitar detección de vitalidad antes de la creación de la prueba |

### Notas

- El token `zelfProof` contiene la prueba facial encriptada y debe almacenarse de forma segura
- Los datos IPFS incluyen metadatos sobre la prueba almacenada y claves de datos públicos
- Se deducen créditos por la creación de la prueba (típicamente 0.84 créditos)
- El `identifier` debe ser solo alfanumérico (sin espacios ni caracteres especiales)
- `publicData` y `metadata` deben contener solo pares clave-valor de string
- La detección de vitalidad puede habilitarse para seguridad adicional
- La protección con contraseña añade una capa extra de seguridad a la prueba
