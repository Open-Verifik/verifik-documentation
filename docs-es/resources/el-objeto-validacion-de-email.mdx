# El Objeto Validación de Email

El objeto Validación de Email representa el proceso de verificación de direcciones de email dentro de tus proyectos Verifik. Este objeto contiene toda la información necesaria para rastrear y gestionar solicitudes de validación de email.

## Atributos

**`client`** - ObjectId - Opcional

La cuenta cliente que posee esta validación de email. Se establece automáticamente basado en tu token de autenticación.

**`project`** - ObjectId - Opcional

El proyecto donde se está realizando esta validación de email. Este es el `_id` generado al crear un proyecto.

**`projectFlow`** - ObjectId - Requerido

El flujo de proyecto donde se está realizando esta validación de email. Este es el `_id` generado al crear un flujo de proyecto.

**`status`** - String - Requerido

El estado actual del proceso de validación de email. Puede ser:

* `"new"` - Solicitud de validación creada pero aún no procesada (por defecto)
* `"sent"` - Email de verificación ha sido enviado al usuario
* `"validated"` - Validación de email completada exitosamente
* `"expired"` - Código de validación ha expirado
* `"failed"` - Validación falló o fue rechazada

**`email`** - String - Requerido

La dirección de email que está siendo validada. Los espacios se eliminan automáticamente y se convierten a minúsculas.

**`type`** - String - Requerido

El tipo de validación de email que se está realizando:

* `"validation"` - Verificación de email estándar
* `"login"` - Verificación de email para login
* `"onboarding"` - Verificación de email para registro de nuevo usuario
* `"oneTimeLink"` - Verificación de enlace de acceso de un solo uso

**`validationMethod`** - String - Requerido

El método usado para la validación de email:

* `"verificationCode"` - Código OTP enviado por email
* `"oneTimeLink"` - Enlace de un solo uso enviado por email

**`verificationCode`** - String - Opcional

El código OTP generado para la verificación de email (solo presente cuando `validationMethod` es `"verificationCode"`).

**`expiresAt`** - Date - Opcional

La fecha y hora de expiración para el código de validación.

**`redirectUrl`** - String - Opcional

URL para redireccionar después de una validación exitosa.

**`webhookUrl`** - String - Opcional

URL de webhook para notificaciones de estado de validación.

**`identityUrl`** - String - Opcional

URL de verificación de identidad para pasos de verificación adicionales.

**`requires2FA`** - Boolean - Opcional

Bandera que indica si se requiere autenticación de dos factores.

**`ipAddress`** - String - Opcional

Dirección IP del usuario realizando la validación.

**`attempts`** - Number - Opcional

Número de intentos de validación realizados.

**`maxAttempts`** - Number - Opcional

Número máximo de intentos de validación permitidos.

**`createdAt`** - Date - Requerido

Timestamp cuando se creó la validación de email.

**`updatedAt`** - Date - Requerido

Timestamp cuando se actualizó por última vez la validación de email.

**`validatedAt`** - Date - Opcional

Timestamp cuando se completó exitosamente la validación de email.

### Características

- **Identificación Única**: Cada validación tiene un ID único para referencia
- **Estados Múltiples**: Soporte para diferentes estados del proceso de validación
- **Métodos de Validación**: Códigos OTP y enlaces de un solo uso
- **Configuración Flexible**: URLs de redirección, webhooks y configuraciones de seguridad
- **Control de Intentos**: Límites de intentos y seguimiento de intentos realizados
- **Timestamps Completos**: Registro de creación, actualización y validación
- **Información de Contexto**: Datos del cliente, proyecto y flujo de proyecto
