### Descripción General

No puedes crear una instancia de un `AppLogin` directamente. Los AppLogins se generan automáticamente cuando el usuario realiza métodos de validación definidos en el **Proyecto > ProjectFlow**. El AppLogin está vinculado al método de validación respectivo utilizado durante el proceso de inicio de sesión.

### Métodos de Validación

Los AppLogins se crean automáticamente a través de los siguientes métodos de validación:

#### Validación por Correo Electrónico

El AppLogin contendrá un campo `emailValidation` que rastrea los detalles de autenticación relacionados con el correo electrónico.

**Pasos:**
1. El usuario recibe un OTP (Contraseña de Un Solo Uso) por correo electrónico
2. El usuario ingresa el OTP en la página de inicio de sesión o lo valida a través de la API [Validar una Validación de Correo Electrónico](/docs-es/resources/validar-una-validacion-email)
3. Una vez verificado, se crea un objeto AppLogin con `type: email`

Ver: [Ejemplo de Acceso por Email](ejemplo-acceso-por-email)

#### Validación por Teléfono

El AppLogin contendrá un campo `phoneValidation` que captura la autenticación telefónica.

**Pasos:**
1. El usuario recibe un OTP por SMS
2. El usuario ingresa el OTP para iniciar sesión o lo valida a través de la API [Validar una Validación de Teléfono](/docs-es/resources/validar-una-validacion-telefono)
3. Una vez que el OTP es validado, se crea un objeto AppLogin con `type: phone`

#### Validación Biométrica

El AppLogin incluirá un campo `biometricValidation` que registra el evento de autenticación biométrica.

**Pasos:**
1. El usuario proporciona datos biométricos (por ejemplo, reconocimiento facial, huella dactilar)
2. Los datos biométricos se validan utilizando detección de vida en la aplicación web o a través de la API [Validar una Validación Biométrica de Inicio de Sesión de Aplicación](validar-una-validacion-biometrica-inicio-sesion-aplicacion)
3. Tras una validación exitosa, se genera un objeto AppLogin con `type: faceLiveness`

### Notas

Dado que los AppLogins se crean automáticamente durante el proceso de interacción con los métodos de inicio de sesión del **ProjectFlow** (correo electrónico, teléfono o biométricos), no necesitas crearlos manualmente. En su lugar, concéntrate en asegurar que los métodos de validación correctos estén implementados en el flujo de inicio de sesión.
