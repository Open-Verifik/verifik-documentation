# El Objeto Validación de Documento

El objeto Validación de Documento representa el proceso de verificación de documentos dentro de tus proyectos Verifik. Este objeto contiene toda la información necesaria para rastrear y gestionar solicitudes de verificación de documentos, incluyendo escaneo de documentos, procesamiento OCR y resultados de validación.

## Atributos

**`appRegistration`** - ObjectId - Opcional

Referencia al objeto AppRegistration asociado con esta validación de documento.

**`client`** - ObjectId - Opcional

Referencia al objeto Client asociado con esta validación de documento.

**`project`** - ObjectId - Opcional

Referencia al objeto Project asociado con esta validación de documento.

**`projectFlow`** - ObjectId - Opcional

Referencia al objeto ProjectFlow asociado con esta validación de documento.

**`status`** - String - Requerido

El estado actual del proceso de validación de documento. Puede ser:

* `"new"` - Solicitud de validación creada pero aún no procesada (por defecto)
* `"pending"` - El documento está siendo procesado
* `"completed"` - Validación de documento completada exitosamente
* `"failed"` - Validación de documento falló
* `"expired"` - La sesión de validación ha expirado

**`documentType`** - String - Requerido

El tipo de documento siendo validado:

* `"id"` - Tarjeta de identidad nacional
* `"passport"` - Pasaporte
* `"driver_license"` - Licencia de conducir
* `"other"` - Otro tipo de documento

**`country`** - String - Requerido

El código de país donde se emitió el documento (ej., "US", "CO", "MX").

**`documentData`** - Object - Opcional

Contiene los datos extraídos del documento:

* `firstName` - Nombre del documento
* `lastName` - Apellido del documento
* `middleName` - Segundo nombre del documento
* `documentNumber` - Número de documento
* `birthDate` - Fecha de nacimiento
* `expirationDate` - Fecha de expiración del documento
* `issueDate` - Fecha de emisión del documento
* `gender` - Género
* `nationality` - Nacionalidad

**`ocrResults`** - Object - Opcional

Contiene los resultados del procesamiento OCR:

* `confidence` - Puntuación de confianza OCR
* `rawText` - Texto extraído en bruto
* `fields` - Resultados de extracción de campos estructurados

**`validationResults`** - Object - Opcional

Contiene los resultados de validación:

* `documentAuthenticity` - Resultado de verificación de autenticidad del documento
* `dataConsistency` - Resultado de verificación de consistencia de datos
* `livenessDetection` - Resultado de detección de vivacidad
* `antiSpoofing` - Resultado de detección anti-spoofing

**`images`** - Object - Opcional

Contiene las imágenes del documento:

* `frontImage` - Imagen del lado frontal (base64)
* `backImage` - Imagen del lado posterior (base64)
* `selfieImage` - Imagen selfie para comparación

**`riskScore`** - Number - Opcional

Puntuación de evaluación de riesgo para la validación.

**`age`** - String - Opcional

Edad calculada desde la fecha de nacimiento.

**`backUrl`** - String - Opcional

URL para redireccionar de vuelta después de la validación.

**`frontUrl`** - String - Opcional

URL para redireccionar al frente después de la validación.

**`createdAt`** - Date - Requerido

Timestamp cuando se creó la validación de documento.

**`updatedAt`** - Date - Requerido

Timestamp cuando se actualizó por última vez la validación de documento.

**`completedAt`** - Date - Opcional

Timestamp cuando se completó exitosamente la validación de documento.

### Características

- **Identificación Única**: Cada validación tiene un ID único para referencia
- **Estados Múltiples**: Soporte para diferentes estados del proceso de validación
- **Tipos de Documento**: Soporte para IDs, pasaportes, licencias de conducir y otros
- **Datos Extraídos**: Información completa extraída del documento
- **Procesamiento OCR**: Resultados de reconocimiento óptico de caracteres
- **Resultados de Validación**: Verificación de autenticidad y consistencia
- **Imágenes**: Imágenes frontales, posteriores y selfies en base64
- **Evaluación de Riesgo**: Puntuación de riesgo para la validación
- **Timestamps Completos**: Registro de creación, actualización y finalización
- **Información de Contexto**: Datos del cliente, proyecto y flujo de proyecto
