# El Objeto Validación de Teléfono

El objeto Validación de Teléfono representa el proceso de verificación de números de teléfono dentro de tus proyectos Verifik. Este objeto contiene toda la información necesaria para rastrear y gestionar solicitudes de verificación de teléfono.

## Atributos

**`client`** - ObjectId - Solo Lectura

La cuenta cliente que posee esta validación de teléfono. Se establece automáticamente basado en tu token de autenticación.

**`project`** - ObjectId - Opcional

El proyecto donde se está realizando esta validación de teléfono. Este es el `_id` generado al crear un proyecto.

**`projectFlow`** - ObjectId - Requerido

El flujo de proyecto donde se está realizando esta validación de teléfono. Este es el `_id` generado al crear un flujo de proyecto.

**`status`** - String - Requerido

El estado actual del proceso de validación de teléfono. Puede ser:

* `"new"` - Solicitud de validación creada pero aún no procesada (por defecto)
* `"sent"` - Código de verificación ha sido enviado al usuario
* `"validated"` - Teléfono ha sido verificado exitosamente
* `"failed"` - Validación de teléfono falló o expiró

**`countryCode`** - String - Requerido

El código de marcación internacional para el número de teléfono (ej., "+1" para US, "+44" para UK, "+57" para Colombia).

**`phone`** - String - Requerido

El número de teléfono que está siendo validado (los espacios se eliminan automáticamente).

**`type`** - String - Requerido

El tipo de validación de teléfono que se está realizando:

* `"validation"` - Validación general de número de teléfono
* `"login"` - Verificación de teléfono durante el login del usuario
* `"onboarding"` - Verificación de teléfono durante el registro del usuario

**`validationMethod`** - String - Requerido

El método usado para la validación de teléfono:

* `"verificationCode"` - Código OTP enviado por SMS
* `"manual"` - Proceso de verificación manual

**`verificationCode`** - String - Opcional

El código OTP generado para la verificación de teléfono (solo presente cuando `validationMethod` es `"verificationCode"`).

**`expiresAt`** - Date - Opcional

La fecha y hora de expiración para el código de validación.

**`redirectUrl`** - String - Opcional

URL para redireccionar después de una validación exitosa.

**`webhookUrl`** - String - Opcional

URL de webhook para notificaciones de estado de validación.

**`identityUrl`** - String - Opcional

URL de verificación de identidad para pasos de verificación adicionales.

**`requires2FA`** - Boolean - Opcional

Bandera que indica si se requiere autenticación de dos factores.

**`ipAddress`** - String - Opcional

Dirección IP del usuario realizando la validación.

**`attempts`** - Number - Opcional

Número de intentos de validación realizados.

**`maxAttempts`** - Number - Opcional

Número máximo de intentos de validación permitidos.

**`createdAt`** - Date - Requerido

Timestamp cuando se creó la validación de teléfono.

**`updatedAt`** - Date - Requerido

Timestamp cuando se actualizó por última vez la validación de teléfono.

**`validatedAt`** - Date - Opcional

Timestamp cuando se completó exitosamente la validación de teléfono.

### Características

- **Identificación Única**: Cada validación tiene un ID único para referencia
- **Estados Múltiples**: Soporte para diferentes estados del proceso de validación
- **Códigos de País**: Soporte completo para códigos de país internacionales
- **Métodos de Validación**: Códigos OTP y verificación manual
- **Configuración Flexible**: URLs de redirección, webhooks y configuraciones de seguridad
- **Control de Intentos**: Límites de intentos y seguimiento de intentos realizados
- **Timestamps Completos**: Registro de creación, actualización y validación
- **Información de Contexto**: Datos del cliente, proyecto y flujo de proyecto
