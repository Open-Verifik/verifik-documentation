# El Objeto Validación Biométrica

El objeto Validación Biométrica representa el proceso de verificación de identidades de usuarios a través de datos biométricos dentro de tus proyectos Verifik. Este objeto contiene toda la información necesaria para rastrear y gestionar solicitudes de verificación biométrica, incluyendo reconocimiento facial y detección de vivacidad.

## Atributos

**`client`** - ObjectId - Solo Lectura

La cuenta cliente que posee esta validación biométrica. Se establece automáticamente basado en tu token de autenticación.

**`project`** - ObjectId - Opcional

El proyecto donde se está realizando esta validación biométrica. Este es el `_id` generado al crear un proyecto.

**`projectFlow`** - ObjectId - Requerido

El flujo de proyecto donde se está realizando esta validación biométrica. Este es el `_id` generado al crear un flujo de proyecto.

**`livenessSession`** - ObjectId - Opcional

La sesión de detección de vivacidad asociada con esta validación biométrica. Esto rastrea el proceso de verificación de vivacidad del usuario.

**`appRegistration`** - ObjectId - Opcional

El registro de aplicación vinculado a esta validación biométrica. Esto conecta la validación a un proceso específico de registro de usuario.

**`status`** - String - Requerido

El estado actual del proceso de validación biométrica. Puede ser:

* `"new"` - Solicitud de validación creada pero aún no procesada (por defecto)
* `"pending"` - La validación está en progreso
* `"completed"` - Validación biométrica completada exitosamente
* `"failed"` - Validación biométrica falló
* `"expired"` - La sesión de validación ha expirado

**`identifier`** - String - Requerido

Un identificador único para el usuario o sesión (ej., email, teléfono, o ID personalizado).

**`type`** - String - Requerido

El tipo de validación biométrica que se está realizando:

* `"validation"` - Validación general de identidad biométrica
* `"login"` - Verificación biométrica durante la autenticación del usuario
* `"onboarding"` - Verificación biométrica durante el registro del usuario

**`expiresAt`** - Date - Opcional

La fecha y hora de expiración para la sesión de validación.

**`redirectUrl`** - String - Opcional

URL para redireccionar después de una validación exitosa.

**`webhookUrl`** - String - Opcional

URL de webhook para notificaciones de estado de validación.

**`requires2FA`** - Boolean - Opcional

Bandera que indica si se requiere autenticación de dos factores.

**`ipAddress`** - String - Opcional

Dirección IP del usuario realizando la validación.

**`sendViaEmail`** - Boolean - Opcional

Bandera para enviar enlace de validación por email.

**`email`** - String - Opcional

Dirección de email para enviar el enlace de validación (requerido si sendViaEmail es true).

**`language`** - String - Opcional

Idioma para plantillas de email (en/es). Por defecto "en".

**`biometricData`** - Object - Opcional

Contiene los datos biométricos y resultados de verificación:

* `faceImage` - Imagen facial codificada en Base64
* `template` - Datos de plantilla biométrica
* `quality` - Puntuación de calidad de imagen
* `livenessScore` - Puntuación de detección de vivacidad

**`verificationResults`** - Object - Opcional

Contiene los resultados de verificación:

* `livenessDetection` - Resultado de detección de vivacidad
* `identityMatch` - Resultado de coincidencia de identidad
* `qualityScore` - Puntuación de calidad general
* `antiSpoofing` - Resultado de detección anti-spoofing

**`riskScore`** - Number - Opcional

Puntuación de evaluación de riesgo para la validación.

**`createdAt`** - Date - Requerido

Timestamp cuando se creó la validación biométrica.

**`updatedAt`** - Date - Requerido

Timestamp cuando se actualizó por última vez la validación biométrica.

**`completedAt`** - Date - Opcional

Timestamp cuando se completó exitosamente la validación biométrica.

### Características

- **Identificación Única**: Cada validación tiene un ID único para referencia
- **Estados Múltiples**: Soporte para diferentes estados del proceso de validación
- **Datos Biométricos**: Imágenes faciales, plantillas y puntuaciones de calidad
- **Detección de Vivacidad**: Prevención de spoofing y detección de fraude
- **Resultados de Verificación**: Coincidencia de identidad y evaluación de riesgo
- **Configuración Flexible**: URLs de redirección, webhooks y configuraciones de seguridad
- **Timestamps Completos**: Registro de creación, actualización y finalización
- **Información de Contexto**: Datos del cliente, proyecto y flujo de proyecto
- **Soporte Multiidioma**: Plantillas de email en español e inglés
