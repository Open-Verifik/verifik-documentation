# El Objeto Webhook

### **Atributos**

***

**client** - `ObjectId` - Requerido\
Referencia al cliente asociado con el webhook. Este es un campo requerido que vincula el webhook a un cliente específico.

***

**projectFlow** - `Array de ObjectId` - Opcional\
Array de referencias a objetos `ProjectFlow`. Si no se proporcionan flujos de proyecto, este campo por defecto es un array vacío.

***

**statistics** - `Object` - Opcional\
Almacena datos estadísticos relacionados con el webhook. Por defecto, este campo es un objeto vacío `{}`.

***

**isActive** - `Boolean` - Requerido\
Indica si el webhook está activo. Este es un campo requerido y debe establecerse como `true` o `false`.

***

**name** - `String` - Requerido\
El nombre del webhook. Este es un campo requerido y debe proporcionarse como string.

***

**url** - `String` - Requerido\
La URL endpoint a la que el webhook enviará notificaciones. Este es un campo requerido y debe proporcionarse como string válido.

***

**description** - `String` - Opcional\
Una breve descripción del webhook. Este es un campo opcional y puede usarse para agregar más contexto o detalles sobre el webhook.

***

**notification** - `Object` - Opcional\
Contiene configuraciones de notificación para el webhook. El objeto tiene los siguientes sub-campos opcionales:

* **email** - `String` - Opcional\
  Dirección de correo electrónico a la que se enviarán las notificaciones.
* **success** - `Boolean` - Opcional\
  Indica si enviar notificaciones en eventos exitosos. Por defecto es `false`.
* **fail** - `Boolean` - Opcional\
  Indica si enviar notificaciones en eventos de falla. Por defecto es `true`.

***

### El Objeto Webhook

```json
{
  "client": "613a4b978482f60fd31bd7b7",
  "projectFlow": ["613a4b978482f60fd31bd7b8", "613a4b978482f60fd31bd7b9"],
  "statistics": {},
  "isActive": true,
  "name": "Webhook de Procesador de Pagos",
  "url": "https://example.com/webhook-endpoint",
  "description": "Este webhook maneja notificaciones de procesamiento de pagos.",
  "notification": {
    "email": "notify@example.com",
    "success": true,
    "fail": false
  }
}
```

### Características

- **Asociación de Cliente**: Vincula webhooks a clientes específicos
- **Integración con Flujos de Proyecto**: Conectar webhooks a múltiples flujos de proyecto
- **Configuraciones de Notificación**: Configurar notificaciones por correo para eventos de éxito/falla
- **Seguimiento de Estadísticas**: Monitorear rendimiento y uso del webhook
- **Configuración Flexible**: Campos opcionales para descripciones personalizadas y configuraciones
