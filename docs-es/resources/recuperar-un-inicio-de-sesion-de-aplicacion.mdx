# Recuperar un Inicio de Sesión de Aplicación

## Endpoint

```
GET https://api.verifik.co/v2/app-logins/{id}
```

Este endpoint recupera un único registro de inicio de sesión de aplicación por su ID, junto con campos poblados opcionales como `emailValidation`, `phoneValidation` o `biometricValidation`.

### Encabezados

| Nombre        | Valor              |
| ------------- | ------------------ |
| Content-Type  | `application/json` |
| Authorization | `Bearer <token>`   |

### Parámetros

| Nombre | Tipo   | Requerido | Descripción                                                 |
| ------ | ------ | --------- | ----------------------------------------------------------- |
| `id`   | String | Sí        | El ID único del registro de inicio de sesión de aplicación que deseas recuperar. |

### Parámetros de Consulta

| Nombre        | Tipo  | Requerido | Descripción                                                                            |
| ------------- | ----- | --------- | -------------------------------------------------------------------------------------- |
| `populates[]` | Array | No        | Un arreglo que especifica qué campos de validación poblar (ej., `biometricValidation`). |

## Características

- **Recuperación de Inicio de Sesión Único**: Obtener información detallada sobre un intento de inicio de sesión específico
- **Población de Validación**: Incluir objetos de validación poblados (email, teléfono, biométrico)
- **Datos Completos de Inicio de Sesión**: Acceder a todos los detalles de inicio de sesión incluyendo estado y marcas de tiempo
- **Múltiples Lenguajes de Programación**: Soporte para JavaScript, Python, PHP y Swift
- **Manejo de Errores**: Respuestas 404 apropiadas para registros de inicio de sesión inexistentes
- **Población Flexible**: Elegir qué campos de validación poblar

## Casos de Uso

- **Auditoría de Inicio de Sesión**: Revisar intentos de inicio de sesión individuales para propósitos de auditoría
- **Análisis de Validación**: Examinar datos de validación detallados para un inicio de sesión específico
- **Investigación de Seguridad**: Investigar actividades de inicio de sesión sospechosas o fallidas
- **Integración de Flujo de Usuario**: Recuperar datos de inicio de sesión para integración con flujos de aplicación
- **Seguimiento de Estado**: Monitorear el estado de intentos de inicio de sesión específicos
