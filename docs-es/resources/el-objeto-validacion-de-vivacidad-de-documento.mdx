# El Objeto Validación de Vivacidad de Documento

El objeto Validación de Vivacidad de Documento representa el resultado de todas las validaciones realizadas en un documento pasado a través de una imagen base64. Este objeto contiene información sobre varias verificaciones anti-spoofing incluyendo detección de Reproducción de Pantalla, Copia Impresa, Sustitución de Retrato y Manipulación Digital.

## Atributos

**`client`** - ObjectId - Requerido

Referencia al cliente que posee este registro de vivacidad de documento.

**`appRegistration`** - ObjectId - Requerido

Referencia al registro de aplicación asociado con esta validación de vivacidad de documento.

**`imageSaved`** - Boolean - Opcional - Por defecto: false

Indica si la imagen del documento procesada ha sido guardada en almacenamiento.

**`imageUrl`** - String - Opcional - Por defecto: ""

URL donde se almacena la imagen del documento procesada (cuando imageSaved es true).

**`status`** - String - Requerido

El estado actual de la validación de vivacidad de documento. Puede ser:

* `"new"` - Solicitud de validación creada pero aún no procesada (por defecto)
* `"processing"` - El documento está siendo analizado
* `"completed"` - Validación completada exitosamente
* `"failed"` - Validación falló

**`validationResults`** - Object - Opcional

Contiene los resultados de varias verificaciones de validación:

* `screenReplay` - Resultado de detección de reproducción de pantalla
* `printedCopy` - Resultado de detección de copia impresa
* `portraitSubstitution` - Resultado de detección de sustitución de retrato
* `digitalManipulation` - Resultado de detección de manipulación digital

**`calibrationSettings`** - Object - Opcional

Contiene las configuraciones de calibración utilizadas para la validación:

* `screenReplayCalibration` - Nivel de calibración: `SOFT`, `REGULAR`, `HARD`
* `printedCopyCalibration` - Nivel de calibración: `SOFT`, `REGULAR`, `HARD`
* `portraitSubstitutionCalibration` - Nivel de calibración: `SOFT`, `REGULAR`, `HARD`

**`validationFlags`** - Object - Opcional

Contiene banderas de configuración de validación:

* `ignoreDocumentCroppedValidation` - Bandera booleana para ignorar recorte de documento
* `ignoreColourLessValidation` - Bandera booleana para ignorar falta de color

**`riskScore`** - Number - Opcional

Puntuación general de evaluación de riesgo para el documento.

**`confidence`** - Number - Opcional

Puntuación de confianza para los resultados de validación.

**`createdAt`** - Date - Requerido

Timestamp cuando se creó la validación de vivacidad de documento.

**`updatedAt`** - Date - Requerido

Timestamp cuando se actualizó por última vez la validación de vivacidad de documento.

**`completedAt`** - Date - Opcional

Timestamp cuando se completó exitosamente la validación.

### Características

- **Identificación Única**: Cada validación tiene un ID único para referencia
- **Estados Múltiples**: Soporte para diferentes estados del proceso de validación
- **Validaciones Anti-Spoofing**: Cuatro tipos de detección de ataques
- **Configuración de Calibración**: Niveles SOFT, REGULAR y HARD
- **Banderas de Validación**: Configuraciones para ignorar recorte y falta de color
- **Puntuaciones Detalladas**: Puntuaciones de confianza para cada validación
- **Almacenamiento de Imagen**: Opción para guardar imágenes en CDN
- **Evaluación de Riesgo**: Puntuación general de riesgo del documento
- **Timestamps Completos**: Registro de creación, actualización y finalización
- **Información de Contexto**: Datos del cliente y registro de aplicación
