# API pública de Verifik — referencia de endpoints

Este documento explica, en lenguaje claro, **qué hace cada consulta** a la API y **cómo probarla** con ejemplos. No necesitas ser programador: puedes copiar los ejemplos en herramientas como Postman o pedirle a alguien técnico que los use tal cual.

---

## Antes de empezar

- **Dirección base (`UrlBase`)**: en Postman suele ser una variable; en los ejemplos de abajo usamos `https://api.verifik.co` solo como **ejemplo**. La dirección real la da Verifik al darte acceso.

- **Autenticación**: muchas APIs piden una **clave** (token) en la cabecera. Si tu colección la usa, deberás sustituir `TU_CLAVE` por la clave que te entreguen.

- **Método**: indica el tipo de operación. Lo habitual es **GET** (solo leer datos) o **POST** (enviar datos en el cuerpo del mensaje).
