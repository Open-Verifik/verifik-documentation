# 🇦🇷 아르헨티나 — RTO

### 엔드포인트

```
GET https://api.verifik.co/v2/ar/rto
```

이 엔드포인트로 번호판 기준 아르헨티나 RTO 검사 기록을 조회합니다. 응답에는 검사 결과, 검사 유형, 검사 및 만료 날짜, 인증서, 확장성 카테고리, 검사 센터 및 알림 유형이 포함됩니다.

### 헤더

| Name          | Value              |
| ------------- | ------------------ |
| Accept        | `application/json` |
| Authorization | `Bearer <token>`   |

### 매개변수

| 이름 | 유형 | 필수 | 설명 |
| ---- | ---- | -------- | ----------- |
| `plate` | string | **Yes** | 메르코수르 형식 `AA-123-BB` (`AA123BB`로 전송, 공백 및 구두점 없음). |

### 응답 필드

| 필드 | 유형 | 설명 |
| ----- | ---- | ----------- |
| `plate` | string | 정규화된 번호판. |
| `totalRecords` | string | 반환된 RTO 기록 수. |
| `inspections` | object[] | RTO 검사 기록. |
| `inspections[].result` | string \| null | 검사 결과. |
| `inspections[].inspectionType` | string \| null | 검사 유형 라벨. |
| `inspections[].inspectionDate` | string \| null | 검사 날짜. |
| `inspections[].expirationDate` | string \| null | 만료 날짜. |
| `inspections[].certificate` | string \| null | 인증서 식별자. |
| `inspections[].scalabilityCategory` | string \| null | 소스에서 반환된 확장성 카테고리. |
| `inspections[].inspectionCenter` | string \| null | 검사 센터 이름. |
| `inspections[].alertType` | string \| null | 가능한 경우 소스 알림 유형. |

### 참고

- 소스가 인식 가능한 일/월/년 값을 반환하면 날짜 필드가 정규화됩니다.
- RTO 기록은 동일 번호판의 여러 기록을 처리하기 위해 `inspections` 배열로 반환됩니다.
