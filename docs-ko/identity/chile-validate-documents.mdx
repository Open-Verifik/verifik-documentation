# 칠레 신분증 유효성 검증

이 엔드포인트는 식별자뿐 아니라 실물 신분증에 **인쇄된 일련번호**를 사용해 칠레 **RUN** 또는 **RUT**의 **유효(비고)** 상태를 확인합니다. 번호만 존재하는지가 아니라 **카드 자체가 유효한지** 확인해야 할 때—**고신뢰 KYC**나 단순 cédula 조회를 넘는 **컴플라이언스** 단계에 쓰세요.

일련번호 **없이** 민정 명의만 확인하려면 **[칠레 신원 확인](/identity/chile)** (`v2/cl/cedula`)을 사용합니다.

### Endpoint

```
https://api.verifik.co/v2/cl/validate/documents
```

**RUN** 또는 **RUT**, **문서 번호**, **인쇄 일련번호**를 보내면 민정·세무 검증 소스에서 **유효성 스타일 상태**를 받습니다. 실물 문서를 확인해야 할 때 사용합니다.

### Headers

| Name | Value |
| --- | --- |
| Accept | `application/json` |
| Authorization | `Bearer <token>` |

### Parameters

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `documentType` | string | Yes | `RUN`(민정) 또는 `RUT`(세무). |
| `documentNumber` | string | Yes | 통합 가이드에 따른 공백 없는 식별자. |
| `serialNumber` | string | Yes | 카드에 인쇄된 일련번호(소스 필수). |

### Request

<Tabs>
  <TabItem value="node" label="Node.js">

```javascript
import axios from "axios";

const { data } = await axios.get("https://api.verifik.co/v2/cl/validate/documents", {
  params: {
    documentType: "RUN",
    documentNumber: "18562530-3",
    serialNumber: "A123456789",
  },
  headers: { Accept: "application/json", Authorization: `Bearer ${process.env.VERIFIK_TOKEN}` },
});
console.log(data);
```

  </TabItem>

### Notes

- `status` 문자열은 상위 레지스트리 표현에 따릅니다. **유효성 지표**로 보고 법적 조언을 대체하지 마세요.
- **`serialNumber`**는 실물에서 수집하세요. 없으면 민정 **Registro Civil** 흐름이 완료되지 않습니다.
